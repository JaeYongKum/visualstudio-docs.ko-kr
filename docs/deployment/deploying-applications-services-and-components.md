# 응용 프로그램, 서비스 및 구성 요소 배포
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

다른 컴퓨터, 장치, 서버 또는 클라우드에 설치할 수 있도록 응용 프로그램, 서비스 또는 구성 요소를 배포할 수 있습니다.  Visual Studio에서 필요한 배포 유형에 적합한 방법을 선택할 수 있습니다.  
  
 다음 표에서는 다양한 배포 시나리오에 대한 설명과 해당 시나리오를 성공적으로 완료하는 방법에 대한 자세한 내용을 볼 수 있는 링크를 보여줍니다.  
  
## 단원 내용  
  
|배포 시나리오|지원 내용|  
|-------------|-----------|  
|**클라우드에 게시:** Visual Studio를 사용하여 Microsoft Azure에 응용 프로그램, 서비스 및 데이터를 배포하여 어디서든 사용하도록 할 수 있습니다.|[Microsoft Azure에 응용 프로그램 게시](http://msdn.microsoft.com/library/windowsazure/ee460772.aspx)|  
|**Windows 스토어 앱에 게시:** Windows 스토어에서 손쉽게 앱을 작성 및 제출하고 전 세계의 고객에게 판매할 수 있습니다.|[Windows 스토어 앱 패키징, 배포 및 쿼리](http://msdn.microsoft.com/library/hh446593\(v=vs.85\).aspx)|  
|**Windows Phone 앱 게시:** Windows Phone 개발자 센터에서 인증을 받기 위해 새 앱이나 기존 앱에 대한 업데이트를 제출할 수 있습니다.|[Windows Phone 앱 게시](http://dev.windowsphone.com/publish)|  
|**ASP.NET 응용 프로그램 또는 서비스 배포:** 다양한 방법으로 ASP.NET 응용 프로그램과 서비스를 배포할 수 있습니다.|[ASP.NET 웹 응용 프로그램 및 서비스 배포](http://www.asp.net/aspnet/overview/deployment)|  
|**LightSwitch 응용 프로그램 또는 서비스 배포:** LightSwitch를 사용하여 응용 프로그램 및 OData 서비스를 만든 후 웹 서버 또는 Microsoft Azure에 배포할 수 있습니다.|[LightSwitch 응용 프로그램 배포](../Topic/Deploying%20LightSwitch%20Applications.md)|  
|**SharePoint용 앱 게시:** Office 스토어 또는 조직 내부용 앱 카탈로그에 SharePoint용 앱을 게시할 수 있습니다.|[Visual Studio를 사용하여 SharePoint용 앱 게시](http://msdn.microsoft.com/library/office/jj220044\(v=office.15\).aspx)|  
|**Office용 앱 게시:** Office 스토어 또는 조직 내부용 앱 카탈로그에 Office용 앱을 게시할 수 있습니다.|[Office용 앱 게시](http://msdn.microsoft.com/library/office/fp123515.aspx)|  
|**WCF 서비스 배포:** 웹 서버에 배포한 WCF RIA 서비스를 다른 응용 프로그램에서 사용할 수 있습니다.|[WCF RIA Services 솔루션 배포](http://msdn.microsoft.com/library/ff426912\(v=vs.91\).aspx)|  
|**OData 서비스 배포:** 웹 서버에 배포한 OData 서비스를 다른 응용 프로그램에서 사용할 수 있습니다.|[OData 서비스 배포](http://msdn.microsoft.com/library/hh973447.aspx)|  
|**데스크톱 응용 프로그램 배포:** ClickOnce 배포를 사용하여 웹 서버 또는 네트워크 파일 공유에 데스크톱 응용 프로그램을 게시할 수 있습니다.  이렇게 하면 사용자가 클릭 한 번으로 응용 프로그램을 설치할 수 있습니다.|[ClickOnce 보안 및 배포](../deployment/clickonce-security-and-deployment.md)|  
|**설치 프로그램 만들기:**  무료로 제공되는 InstallShield Limited Edition을 사용하여 설치 프로그램을 만들 수 있습니다.|[InstallShield Limited Edition](../deployment/installshield-limited-edition.md)|  
|**기존 설치 프로그램 유지 관리:** Visual Studio 설치 관리자 프로젝트 확장을 설치하면 이전 버전 Visual Studio에서 만든 설치 프로그램을 계속 사용할 수 있습니다.|[Visual Studio 설치 관리자 프로젝트 확장](http://blogs.msdn.com/b/visualstudio/archive/2014/04/17/visual-studio-installer-projects-extension.aspx)<br /><br /> 설치 관리자 프로젝트의 설명서는 [Visual Studio 설치 관리자 배포](http://msdn.microsoft.com/library/2kt85ked\(v=vs.100\).aspx)에서 제공됩니다.|  
|**Visual C\+\+ 응용 프로그램 배포:** 중앙 배포, 로컬 배포 또는 정적 연결을 사용하여 응용 프로그램과 함께 Visual C\+\+ 런타임을 배포할 수 있습니다.|[네이티브 데스크톱 응용 프로그램 배포\(Visual C\+\+\)](http://msdn.microsoft.com/library/zebw5zk9.aspx)|  
|**테스트를 위해 응용 프로그램 배포:**응용 프로그램을 가상 환경에 배포하여 더 정교한 개발 및 테스트를 지원할 수 있습니다.|[랩 환경에서 테스트](/devops-test-docs/test/test-on-a-lab-environment)|  
|**필수 구성 요소 설치:** 일반 설치 관리자를 구성하여 데스크톱 응용 프로그램의 필수 구성 요소\(부트스트래퍼\)를 설치할 수 있습니다.|[응용 프로그램 배포 필수 구성 요소](../deployment/application-deployment-prerequisites.md)|
