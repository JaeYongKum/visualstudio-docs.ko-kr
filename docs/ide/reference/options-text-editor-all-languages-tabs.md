# 옵션, 텍스트 편집기, 모든 언어, 탭
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

이 대화 상자에서는 코드 편집기의 기본 동작을 변경할 수 있습니다.  이러한 설정은 HTML 디자이너의 소스 뷰와 같은 코드 편집기 기반의 다른 편집기에도 영향을 미칩니다.  이러한 옵션을 표시하려면 **도구** 메뉴에서 **옵션**을 선택합니다.  **텍스트 편집기** 폴더 내에서 **모든 언어** 하위 폴더를 확장한 다음 **탭**을 선택합니다.  
  
> [!CAUTION]
>  이 페이지에서는 모든 개발 언어에 대해 기본 옵션을 설정합니다.  이 대화 상자에서 옵션을 다시 설정하면 모든 언어의 탭 옵션이 여기서 선택한 항목으로 설정됩니다.  한 언어에 대한 텍스트 편집기 옵션을 변경하려면 해당 언어의 하위 폴더를 확장한 다음 해당 옵션 페이지를 선택합니다.  
  
 특정 프로그래밍 언어에 대한 탭 옵션 페이지에서 각각 다른 설정을 선택하면, 다른 **들여쓰기** 옵션에 대해 "개별 텍스트 형식에 대한 들여쓰기 설정이 서로 충돌합니다."라는 메시지가 표시되고 다른 **탭** 옵션에 대해 "개별 텍스트 형식에 대한 탭 설정이 서로 충돌합니다."라는 메시지가 표시됩니다.  예를 들어, Visual Basic에 대해 **스마트 들여쓰기** 옵션이 선택되었지만 Visual C\+\+에 대해서는 **블록 들여쓰기**가 선택된 경우 이러한 메시지가 표시됩니다.  
  
> [!NOTE]
>  표시되는 대화 상자와 메뉴 명령은 활성 설정이나 버전에 따라 도움말에서 설명하는 것과 다를 수 있습니다.  설정을 변경하려면 **도구** 메뉴에서 **설정 가져오기 및 내보내기**를 선택합니다.  자세한 내용은 [Customizing Development Settings in Visual Studio](http://msdn.microsoft.com/ko-kr/22c4debb-4e31-47a8-8f19-16f328d7dcd3)을 참조하십시오.  
  
## 들여쓰기  
 없음  
 선택할 경우 새 줄이 들여쓰기되지 않습니다.  삽입 지점이 새 줄의 첫 번째 열에 배치됩니다.  
  
 블록  
 선택할 경우 새 줄이 자동으로 들여쓰기됩니다.  삽입 지점은 이전 줄과 동일한 시작 지점에 배치됩니다.  
  
 스마트  
 선택할 경우 새 줄이 기타 코드 서식 설정 및 사용 중인 개발 언어의 IntelliSense 규칙에 따라 코드 컨텍스트에 맞게 배치됩니다.  이 옵션은 모든 개발 언어에서 사용할 수 있는 것은 아닙니다.  
  
 예를 들어, 여는 중괄호\({\)와 닫는 중괄호\(}\) 사이에 포함된 줄은 정렬된 중괄호 위치에서 자동으로 탭 정지 하나만큼 추가로 들여쓰기될 수 있습니다.  
  
## 탭  
 탭 크기  
 탭 정지 간의 거리\(공백 수\)를 설정합니다.  기본값은 4입니다.  
  
 들여쓰기 크기  
 자동 들여쓰기의 크기\(공백 수\)를 설정합니다.  기본값은 4입니다.  탭 문자나 공백 문자가 삽입되거나 둘 다 삽입되어 지정된 크기를 채웁니다.  
  
 공백 삽입  
 이 옵션을 선택하고 들여쓰기 작업을 수행하면 탭 문자가 아닌 공백 문자만 삽입됩니다.  예를 들어, **들여쓰기 크기**가 5로 설정되어 있으면 Tab 키를 누르거나 **서식** 도구 모음에서 **들여쓰기** 단추를 누를 때마다 공백 문자 5개가 삽입됩니다.  
  
 탭 유지  
 이 옵션을 선택하고 들여쓰기 작업을 수행하면 가능한 최대 개수만큼 탭 문자가 삽입됩니다.  각 탭 문자는 **탭 크기**에 지정된 공백 수만큼 채웁니다.  **들여쓰기 크기**가 **탭 크기**의 짝수 배수가 아니면 공백 문자가 추가되어 차이를 메꿉니다.
