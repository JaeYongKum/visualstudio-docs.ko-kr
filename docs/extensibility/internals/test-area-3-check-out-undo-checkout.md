# <a name="test-area-3-check-outundo-checkout"></a>테스트 영역 3: 체크 아웃/체크 아웃 취소
이 소스 제어 플러그 인 테스트 기능은 버전 저장소를 통해 항목을 편집 및 되돌리기는 **체크 아웃** 및 **체크 아웃 취소** 명령입니다.  
  
 **체크 아웃**: 표시 버전 저장소에 있는 항목 체크 아웃 된 대로 로컬 사본을 수정 읽기/쓰기입니다.  
  
 **체크 아웃 취소**: 체크 인 된 대로 버전 저장소에 있는 항목을 표시, 로컬 복사본을 체크 아웃 (옵션)에 따라 하기 전의 상태로 되돌립니다.  
  
## <a name="command-menu-access"></a>명령 메뉴 액세스  
 다음 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 통합된 개발 환경 메뉴 경로 테스트 사례에서 사용 됩니다.  
  
##### <a name="check-out"></a>체크 아웃:  
  
-   **파일**, **소스 제어**, **체크 아웃**합니다.  
  
-   **파일**, **체크 아웃**합니다.  
  
-   바로 가기 메뉴를 **체크 아웃**합니다.  
  
-   체크 아웃을 취소: **파일**, **소스 제어**, **체크 아웃 취소**합니다.  
  
## <a name="common-expected-behavior"></a>예상 되는 일반적인 동작  
  
-   체크아웃 작업 후 대상 파일 및/또는 폴더의 버전 저장소에 체크 아웃 된 상태로 표시 됩니다.  
  
-   버전 저장소 특성을 올바른 사용자에 게 체크 아웃 합니다.  
  
-   체크 아웃의 날짜 및 시간 (사용자의 설정)에 따라 올바릅니다.  
  
## <a name="test-cases"></a>테스트 사례  
 체크 아웃을 체크 아웃/취소 테스트 영역에 대 한 특정 테스트 사례는 다음과 같습니다.

### <a name="case-3b-disconnected-check-out"></a>사례 3b: 체크 아웃 연결 끊김  
 연결이 끊어진된 모드에서 작업할 일정 수준의 지속적인된 소스 제어 지원 버전 저장소에 직접 연결 되지 않은 사용자가 있습니다. 이 작업은 로컬로 참여 한 솔루션 및 프로젝트에 대 한 모든 관련 정보를 캐시 하 여 수행 됩니다.  
  
 단독 체크 아웃 작업 소스 제어 저장소에 연결 된 동안에 발생할 수 있습니다. 공유 체크 아웃 작업에 연결 되었거나 연결이 있는지 여부를 언제 든 지 발생할 수 있습니다. 따라서 버전 저장소에서 연결이 끊기면는 **체크 아웃 공유** (CO) 명령을 사용할 수 있습니다. 연결이 끊긴 동안 **체크 아웃 취소** 사용할 수 없습니다. 변경 내용을 사용자가을 바꾸려면 이전 버전을 검색할 수 없습니다.  
  
 사용자는 버전에 다시 연결할 때 모든 참여 한 솔루션의 체크 아웃 상태 저장 및 프로젝트 동기화 됩니다. 이 사용자가 수행 하는 체크 아웃에 대 한 저장소에 필요한 업데이트를 수행 합니다. 동기화 조치가 발생 되 면 사용자가 (연결) 일반적인 방법으로 작업을 계속할 수 있습니다.  
  
#### <a name="expected-behavior"></a>예상 되는 동작  
  
-   사용할 수 없는 **단독 체크 아웃** 버전 저장소에서 연결이 끊어진 동안 명령입니다.  
  
-   사용할 수 없는 **체크 아웃 취소** 버전 저장소에서 연결이 끊어진 동안 명령입니다.  
  
-   **공유 체크 아웃** 명령은 작동 합니다.  
  
|작업|테스트 단계|예상된 결과를 확인합니다|  
|------------|----------------|--------------------------------|  
|연결이 끊긴 동안 파일을 체크 아웃할 다음 동기화에 대 한 연결|1.  소스 제어 변경 대화 상자를 사용 하 여 제어 프로젝트 연결을 끊습니다 (**파일**, **소스 제어**, **소스 제어 변경**l).<br />2.  파일을 체크 아웃 합니다.<br />3.  체크 아웃 (연결 끊김) 경고 대화 상자에서를 클릭 합니다.<br />4.  파일을 편집합니다.<br />5.  소스 제어 변경 대화 상자를 사용 하 여 연결 합니다.<br />6.  편집된 된 파일의 최신 버전을 가져옵니다.|예상 되는 일반적인 동작|  
  
### <a name="case-3c-query-editquery-save-qeqs"></a>3c 사례: 쿼리 편집/쿼리 저장 (QEQS)  
 소스 제어에서 항목은 편집, 변경 내용에 대 한 추적 및 해당 파일을 관리 하는 데 도움을 주려면 쉽게 저장 합니다. "체크 인 된" 제어 항목을 편집할 때 QEQS 시도한 편집을 차단 하 고 편집할 파일을 체크 아웃 하는 사용자에 게 요청 합니다. 에 따라 **도구**, **옵션** 설정, 사용자가 파일을 편집 하려면 우선 out 강제로 확인 하나 또는 메모리에 복사본 편집 하 고 나중에 체크 아웃을 사용할 수 있습니다. 하는 경우 사용자의 **도구**, **옵션** 를 체크 아웃 대화 상자를 표시 하 고 방금 체크 아웃 설정을 설정 하지 않은 다음 사용자가 자신의 편집, 파일이 자동으로 체크 아웃, 가능 합니다.  
  
#### <a name="expected-behavior"></a>예상 되는 동작  
  
-   체크아웃 작업 후 대상 파일 및/또는 폴더의 버전 저장소에 체크 아웃 된 상태로 표시 됩니다.  
  
-   버전 저장소 특성을 올바른 사용자에 게 체크 아웃 합니다.  
  
-   체크 아웃의 날짜 및 시간 (사용자의 설정)에 따라 올바릅니다.  
  
-   대상 파일 또는 폴더의 로컬 복사본은 쓰기 가능 합니다.  
  
|작업|테스트 단계|예상된 결과를 확인합니다|  
|------------|----------------|--------------------------------|  
|체크 인 텍스트 파일 편집|1.  텍스트 파일을 포함 하는 새 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  설정 **도구**, **옵션**, **소스 제어**, **파일을 디스크에서 읽기 전용으로 설정 하는 동안 편집할 수** 이 옵션을 선택 취소 합니다.<br />4.  설정 **도구**, **옵션**, **소스 제어**, **체크 아웃 확인** 에 **파일을 편집한체크인될때** 콤보 상자입니다.<br />5.  설정 **도구**, **옵션**, **소스 제어**, **체크 아웃 확인** 에 **체크 인 될 때 파일은** 콤보 상자입니다.<br />6.  편집기에서 텍스트 파일을 열고, 파일에 새 텍스트를 입력 하려고 합니다. 이 단계에 성공 하면 다음 단계를 계속 합니다.<br />7.  클릭 **취소** 에 **편집 하기 위해 체크 아웃** 대화 상자. 이 단계에 성공 하면 다음 단계를 계속 합니다.<br />8.  설정 **도구**, **옵션**, **소스 제어**, **파일을 디스크에서 읽기 전용으로 설정 하는 동안 편집할 수** 에 선택 합니다.<br />9. 편집기에서 프로젝트 파일을 열고, 파일에 새 텍스트를 입력 하려고 합니다. 이 단계에 성공 하면 다음 단계를 계속 합니다.<br />10. 클릭 **편집** 에 **편집 하기 위해 체크 아웃** 대화 상자. 이 단계에 성공 하면 다음 단계를 계속 합니다.<br />11. 텍스트 파일을 편집 하 고 저장 하려고 합니다.|`Result of step 6:`<br /><br /> 편집 대화 상자가 나타납니다에 대 한 체크 아웃 합니다.<br /><br /> `Result of step 7:`<br /><br /> 파일 변경 되지 않습니다.<br /><br /> `Result of step 9:`<br /><br /> 편집 대화 상자가 나타납니다에 대 한 체크 아웃 합니다.<br /><br /> `Result of step 10:`<br /><br /> 메모리에 있는 프로젝트 파일을 편집할 수 있습니다.<br /><br /> `Result of step 11:`<br /><br /> 저장을 체크 아웃에 저장 대화 상자가 나타납니다.|  
|체크 인 된 솔루션 파일을 편집 합니다.|반복 이전에 설명 된 대로 단계를 테스트 하지만 텍스트 파일을 수정 하는 대신 솔루션 속성을 변경 하 여 솔루션을 수정 합니다.|이전 테스트와 동일|  
|체크 인 한 프로젝트 파일 편집|반복 이전에 설명 된 대로 단계를 테스트 하지만 텍스트 파일을 수정 하는 대신 프로젝트 속성을 변경 하 여 프로젝트를 수정 합니다.|이전 테스트와 동일 합니다.|  
  
### <a name="case-3d-silent-check-out"></a>자동 체크 아웃 3d 대/소문자:  
 이 하위 영역에서는 체크 시나리오 아웃 여기서는 **체크 아웃** 사용자 당 대화 상자가 나타나지 않으면 **도구**, **옵션**, **소스 제어 설정** .  
  
#### <a name="expected-behavior"></a>예상 되는 동작  
  
-   체크아웃 작업 후 대상 파일 및/또는 폴더의 버전 저장소에 체크 아웃 된 상태로 표시 됩니다.  
  
-   버전 저장소 특성을 올바른 사용자에 게 체크 아웃 합니다.  
  
-   체크 아웃의 날짜 및 시간 (사용자의 설정)에 따라 올바릅니다.  
  
-   대상 파일 또는 폴더의 로컬 복사본은 쓰기 가능 합니다.  
  
|작업|테스트 단계|예상된 결과를 확인합니다|  
|------------|----------------|--------------------------------|  
|파일에 대 한 자동 체크 아웃|1.  설정 **도구**, **옵션**, **소스 제어** 를 **편집에 자동으로 파일 체크 아웃**합니다.<br />2.  파일을 새 프로젝트를 만듭니다.<br />3.  소스 제어에 솔루션을 추가 합니다.<br />4.  파일을 체크 아웃 합니다.|파일을 체크 아웃 자동으로 (UI 없음).|  
|프로젝트에 대 한 자동 체크 아웃|1.  설정 **도구**, **옵션**, **소스 제어** 를 **편집에 자동으로 파일 체크 아웃**합니다.<br />2.  새 프로젝트를 만듭니다.<br />3.  소스 제어에 솔루션을 추가 합니다.<br />4.  프로젝트 확인해 보세요.|파일을 체크 아웃 자동으로 (UI 없음).|  
  
### <a name="case-3e-undo-check-out"></a>사례 3e: 체크 아웃 실행 취소  
 **체크 아웃 실행 취소** 취소 파일의 체크 아웃 상태를 파일에 대 한 변경 내용 체크 인을 방지 하는 데 사용 됩니다.  
  
#### <a name="expected-behavior"></a>예상 되는 동작  
  
-   기본값은 사용자의 기반 **로컬 버전 체크 아웃** 설정 합니다. 사용자가 로컬 버전을 체크 아웃을 선택한 경우 항상 체크 아웃 한 버전으로 되돌리려면 체크 아웃을 취소에 대 한 기본값은입니다.  
  
-   실행 취소 아이콘에 대 한 동의 시 **솔루션 탐색기** 영향에 대 한 업데이트 파일과 항목에서 제거 되는 **보류 중인 체크 인** 창.  
  
|작업|테스트 단계|예상된 결과를 확인합니다|  
|------------|----------------|--------------------------------|  
|단독으로 체크 아웃 하는 단일 파일이 체크 아웃 취소|1.  클라이언트 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  단독 체크 아웃 한 파일입니다.<br />4.  파일을 수정 합니다.<br />5.  체크 아웃 취소 (**파일**, **소스 제어**, **체크 아웃 취소**).|예상 되는 일반적인 동작입니다.|  
|체크 아웃 공유 하는 단일 파일이 체크 아웃 취소|1.  클라이언트 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  파일 공유 확인해 보세요.<br />4.  파일을 수정 합니다.<br />5.  체크 아웃 취소 (**파일**, **소스 제어**, **체크 아웃 취소**).|예상 되는 일반적인 동작입니다.|  
|파일을 프로젝트에 추가한 후 프로젝트를 체크 아웃 취소|1.  새 프로젝트를 만들고 소스 제어에 추가 합니다.<br />2.  프로젝트 확인해 보세요.<br />3.  파일을 프로젝트에 추가 합니다.<br />4.  프로젝트의 체크 아웃을 취소 합니다.|추가 된 파일은 솔루션 탐색기에서 프로젝트에서 제거 됩니다.<br /><br /> 프로젝트가 더 이상 체크 아웃 합니다.|  
|프로젝트에서 파일을 삭제 한 후 프로젝트를 체크 아웃 취소|1.  새 프로젝트를 만들고 소스 제어에 추가 합니다.<br />2.  프로젝트 확인해 보세요.<br />3.  프로젝트에서 파일을 삭제 합니다.<br />4.  프로젝트의 체크 아웃을 취소 합니다.|삭제 된 파일이 솔루션 탐색기에서 프로젝트 아래에 나타납니다.<br /><br /> 프로젝트가 더 이상 체크 아웃 합니다.|
