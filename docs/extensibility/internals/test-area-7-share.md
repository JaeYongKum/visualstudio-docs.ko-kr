# <a name="test-area-7-share"></a>테스트 영역 7: 공유
이 테스트 기능은 공유 항목을 통해 위치 간에 **공유** 명령입니다.  
  
 hhare 작업이 파일 및 소스 제어 파일 계층 구조 내에서 둘 이상의 위치 간에 항목 폴더의 명백한 중복 되었습니다. 복제 서버에서 실제로 발생 하지 않습니다 되지만 사용자가 두 개 이상의 지정 된 위치에서 동일한 파일을 참조 하십시오. 공유 항목 중 하나에 변경 될 때마다 이러한 변경 내용이 다른 모든 공유 위치에 표시 합니다.  
  
 폴더에 공유에 소스 제어에서 하나 이상의 파일이 있는 폴더를 선택 하는 경우 작동 합니다. 다음과 같은 공유 명령을 비활성화 됩니다.  
  
-   선택한 폴더에 빈 폴더 경우.  
  
-   실제 폴더 올바르지만 없는 소스 제어 파일을 포함 합니다.  
  
-   경우 소스 제어에서 파일에는 인지 여부를는 가상 폴더입니다.  
  
-   원격 사이트 웹 프로젝트인 경우.  
  
## <a name="command-menu-access"></a>명령 메뉴 액세스  
 다음 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 통합된 개발 환경 메뉴 경로 테스트 사례에서 사용 됩니다.  
  
 공유: **파일**->**소스 제어**->**공유**합니다.  
  
## <a name="expected-behavior"></a>예상 되는 동작  
  
-   공유 파일 공유 위치에 나타납니다.  
  
-   소스 제어 버전 저장소 기록에 표시 된 파일 공유는 볼 수 있습니다.  
  
-   공유 파일을 편집 하는 파일의 두 위치를 편집 합니다.  
  
## <a name="test-cases"></a>테스트 사례  
 공유 테스트 영역에 대 한 특정 테스트 사례는 다음과 같습니다.  
  
|작업|테스트 단계|예상된 결과를 확인합니다|  
|------------|----------------|--------------------------------|  
|다른 로드 된 프로젝트를 소스 제어에서 로드 한 프로젝트에서 파일 공유|1.  새 프로젝트를 만듭니다.<br />2.  두 번째 프로젝트를 솔루션에 추가 합니다.<br />3.  첫 번째 프로젝트에 없는 이름 사용 하 여 두 번째 프로젝트에 파일을 만듭니다.<br />4.  소스 제어에 솔루션을 추가 합니다.<br />5.  첫 번째 프로젝트를 선택 합니다.<br />6.  열기 **공유** 대화 상자 (**파일** -> **소스 제어** -> **공유**).<br />7.  첫 번째 프로젝트에 대 한 두 번째 프로젝트 파일을 공유 합니다.<br />8.  수락 **체크 아웃** 메시지가 표시 되 면 합니다.|예상 되는 일반적인 동작입니다.|
