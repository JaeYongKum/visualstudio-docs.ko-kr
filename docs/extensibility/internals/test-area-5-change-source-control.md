# 소스 제어를 변경 하는 테스트 영역 5:
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

이 소스 제어 플러그 인 테스트 기능은 통한 소스 제어 변경는 **소스 제어 변경** 명령입니다.  
  
 **소스 제어 변경** 명령은 사용자에 대 한 네 가지 기본 기능을 제공 합니다.  
  
-   **바인딩:**  
  
     설정 하거나 소스 제어 링크 솔루션\/프로젝트와 버전 저장소를 다시 설정 하는 사용자 수 있습니다.  
  
-   **바인딩 해제 합니다.**  
  
     각 연결 단위로 소스 제어에서 프로젝트\/솔루션을 제거합니다.  
  
-   **연결\/분리 합니다.**  
  
 영역 3에서 다룬 제어 솔루션의 설정\/해제 연결 또는 오프 라인 상태입니다. 자세한 내용은 [테스트 영역 3: 체크 아웃\/체크 아웃 취소](../../extensibility/internals/test-area-3-check-out-undo-checkout.md)을 참조하세요.  
  
## 명령 메뉴 액세스  
 다음 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 통합된 개발 환경 메뉴 경로 테스트 사례에서 사용 됩니다.  
  
 소스 제어 변경:**파일**, **소스 제어**, **소스 제어 변경**합니다.  
  
## 테스트 사례  
 다음은 특정 테스트 사례에 대 한는 **소스 제어 변경** 테스트 영역 명령입니다.  
  
### 5a case: 바인딩  
 바인딩 사용자를 선택한 프로젝트 및 솔루션에 소스 코드 제어 정보를 추가 하 고 있습니다. 사용자는 일반적으로 추가 될 이들은 소스 제어에서 프로젝트를 식별 하 라는 메시지가 표시 됩니다. 사용자 \(소스 제어에 추가 된 대비\)이이 작업의 일부분으로 소스 제어에서 새 프로젝트를 만들 수 없습니다.  
  
|작업|테스트 단계|예상된 결과 확인 하려면|  
|--------|------------|-------------------|  
|비어 있는 위치에 바인딩|1.  프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  열기 **소스 제어 변경** 대화 상자 \(**파일**, **소스 제어**, **소스 제어 변경**\).<br />4.  클릭 **바인딩 해제**합니다.<br />5.  표시 되는 경우 경고 대화 상자를 수락 합니다.<br />6.  모든 항목을 선택 합니다.<br />7.  클릭 **바인딩할**합니다.<br />8.  소스 제어 저장소에서 빈 위치를 찾습니다.<br />9. 클릭 **확인** 를 닫으려면는 **소스 제어 변경** 대화 상자입니다.<br />10. 클릭 **계속이 바인딩** 확인 대화 상자에서 합니다.<br />11. 클릭 **확인** 경고 대화 상자가 표시 되는 경우에 있습니다.<br />12. 모두 체크 인 합니다. 이 단계에 성공 하면 다음 단계를 계속 합니다.<br />13. 새 위치에 소스 제어에서 솔루션을 엽니다.|`Result from Step 12:`<br /><br /> 솔루션 및 프로젝트에 바인딩된 및 버전 저장소에 새 대상에 기록 합니다.<br /><br /> 솔루션 및 프로젝트 파일에 대 한 체크 인 됩니다.<br /><br /> 버전 저장소에 대 한 프로젝트 계층 구조에는 디스크에 있는 프로젝트의 폴더 계층 구조와 일치합니다.<br /><br /> `Result from Step 13:`<br /><br /> 프로젝트 항목을 모두 다운로드 됩니다.|  
|클라이언트와 동기화 된 위치에 바인딩|1.  프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  버전 저장소에 솔루션 및 프로젝트의 복제본 만들기 \(공유 하 고 사용 하는 경우 분기 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]\).<br />4.  열기 **소스 제어 변경** 대화 상자 \(**파일**, **소스 제어**, **소스 제어 변경**\).<br />5.  모든 바인딩을 해제 합니다.<br />6.  클릭 **확인** 를 닫으려면 **소스 제어 변경** 대화 상자입니다.<br />7.  다시 **소스 제어 변경** 대화 상자입니다.<br />8.  모두 선택 합니다.<br />9. 클릭 **바인딩할**합니다.<br />10. \(3 단계\)에서 솔루션 및 프로젝트의 분기 된 위치로 이동 합니다.<br />11. 클릭 **확인** 를 닫으려면는 **소스 제어 변경** 대화 상자입니다.<br />12. 모든 항목에 대 한 최신 재귀적으로 가져옵니다.|Get get 전과 동일 하 게 된 후 콘텐츠를 파일입니다.|  
|클라이언트와 동기화 되지 않은 위치에 바인딩|1.  프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  버전 저장소에 솔루션 및 프로젝트의 복제본 만들기 \(공유 하 고 사용 하는 경우 분기 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]\).<br />4.  버전 저장소에 분기 된 프로젝트의 파일을 수정 합니다.<br />5.  열기 **소스 제어 변경** 대화 상자 \(**파일**, **소스 제어**, **소스 제어 변경**\).<br />6.  모든 바인딩을 해제 합니다.<br />7.  클릭 **확인** 를 닫으려면 **소스 제어 변경** 대화 상자입니다.<br />8.  다시 **소스 제어 변경** 대화 상자입니다.<br />9. 모두 선택 합니다.<br />10. 클릭 **바인딩할**합니다.<br />11. 분기 솔루션 및 프로젝트에 대 한 위치를 찾습니다.<br />12. 클릭 **확인** 를 닫으려면는 **소스 제어 변경** 대화 상자입니다.<br />13. 표시 되는 경우 경고 대화 상자를 수락 합니다.<br />14. 모든 항목에 대 한 최신 재귀를 가져옵니다.|4 단계에서에서 수정 된 파일을 로컬로 수정 됩니다.|  
|소스 제어 되지 바인딩 솔루션|1.  소스 제어에 빈 폴더를 만듭니다.<br />2.  클라이언트 프로젝트를 만듭니다.<br />3.  열기 **소스 제어 변경** 대화 상자 \(**파일**, **소스 제어**, **소스 제어 변경**\).<br />4.  소스 제어에서 빈 위치에 솔루션을 바인딩하십시오.<br />5.  클릭 **확인** 를 닫으려면는 **소스 제어 변경** 대화 상자입니다.<br />6.  클릭 **계속이 바인딩** 확인 대화 상자에서 합니다.<br />7.  클릭 **확인** 경고 대화 상자가 표시 되는 경우에 있습니다.|소스 제어에 솔루션 추가 됩니다.<br /><br /> 솔루션 및 프로젝트를 체크 아웃 됩니다.|  
|취소 바인딩|1.  프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  소스 제어 변경 대화 상자를 엽니다.<br />4.  모든 바인딩을 해제 합니다.<br />5.  클릭 **확인** 단추 대화 상자를 닫습니다. 이 단계에 성공 하면 다음 단계를 계속 합니다.<br />6.  다시는 **소스 제어 변경** 대화 상자입니다.<br />7.  관련 되지 않은 위치에 바인딩하십시오.<br />8.  **취소**를 클릭합니다.|`Result from Step 5:`<br /><br /> 솔루션은 소스 제어에서 더 이상<br /><br /> `Result from Step 8:`<br /><br /> 솔루션은 여전히 하지 아래에서 소스 제어 합니다.|  
  
### 5b case: 바인딩 해제  
 프로젝트 및 솔루션 제거 소스 코드 제어 정보를 바인딩 해제 합니다. 영향을 받는 프로젝트 및 솔루션의 기반으로 사용자 선택 및 소스 제어에 추가 된 항목의 조합 합니다.  
  
|작업|테스트 단계|예상된 결과 확인 하려면|  
|--------|------------|-------------------|  
|하나의 파일 시스템 또는 로컬 IIS 웹 프로젝트와 하나의 클라이언트 프로젝트에 포함 된 솔루션을 바인딩 해제|1.  파일 시스템 또는 로컬 IIS 웹 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  새 클라이언트 프로젝트를 솔루션에 추가 합니다.<br />4.  메시지가 표시 되 면의 체크 아웃 솔루션을 수락 합니다.<br />5.  열기는 **소스 제어 변경** 대화 상자입니다.<br />6.  클릭 **바인딩 해제**합니다.<br />7.  클릭 **확인** 대화 상자를 닫습니다.<br />8.  솔루션, 프로젝트, 솔루션 항목, 프로젝트 항목 확인 하려고 합니다.|솔루션 및 프로젝트 소스 제어 하지 않습니다.<br /><br /> 소스 제어 메뉴 명령을 표시 되지 않습니다.|  
|취소 바인딩 해제|1.  프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  열기는 **소스 제어 변경** 대화 상자입니다.<br />4.  클릭 **모두 언바인딩**합니다.<br />5.  **취소**를 클릭합니다.|소스 제어에서 솔루션은입니다.|  
  
### 사례 5 c: 다시 바인딩  
 Rebind은 바인딩 및 바인딩 해제의 조합 단순히\-소스 제어에서 이전에 되 고 바인딩된 없습니다 된 프로젝트\/솔루션을 다시 바인딩해야 하는 과정입니다.  
  
|작업|테스트 단계|예상된 결과 확인 하려면|  
|--------|------------|-------------------|  
|솔루션과 프로젝트를 닫지 않고 다시 바인딩하려면는 **소스 제어 변경** 대화 상자|1.  프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  열기는 **소스 제어 변경** 대화 상자입니다.<br />4.  클릭 **바인딩 해제**합니다.<br />5.  모든 행을 선택 합니다.<br />6.  클릭 **바인딩할**합니다.<br />7.  클릭 **확인** 를 닫으려면는 **소스 제어 변경** 대화 상자입니다.<br />8.  메시지가 표시 되 면 체크 아웃을 수락 합니다.|솔루션 및 프로젝트 소스 제어 됩니다.|  
|프로젝트만 닫지 않고 다시 바인딩하려면 **소스 제어 변경** 대화 상자|1.  프로젝트를 만듭니다.<br />2.  프로젝트에만 사용 하 여 소스 제어에 추가 \(File\-\>는 소스 제어에 소스 제어에 선택한 프로젝트 추가\-\>.<br />3.  소스 제어 변경 대화 상자를 엽니다.<br />4.  프로젝트에만 바인딩 해제 합니다.<br />5.  프로젝트만을 바인딩하십시오.|솔루션에 제어 되지 않은 상태로 유지 됩니다.<br /><br /> 프로젝트에 제어 된 상태로 유지 됩니다.|  
|솔루션만 닫지 않고 다시 바인딩하려면 **소스 제어 변경** 대화 상자|1.  프로젝트를 만듭니다.<br />2.  사용 하 여 소스 제어에 솔루션만 추가 \(**파일**, **소스 제어**, **소스 제어에 선택한 프로젝트 추가**합니다.<br />3.  열기는 **소스 제어 변경** 대화 상자입니다.<br />4.  솔루션만을 바인딩 해제 \(닫지 마십시오 **소스 제어 변경** 대화 상자.\)<br />5.  솔루션만을 바인딩하십시오.<br />6.  **확인**을 클릭하여 대화 상자를 닫습니다.<br />7.  솔루션 및 확인 솔루션 항목 \(있는 경우.\)|솔루션에 제어 된 상태로 유지 됩니다.<br /><br /> 프로젝트에 제어 되지 않은 상태로 유지 됩니다.|  
|같은 디렉터리에만 솔루션\/프로젝트를 다시 바인딩하십시오.|1.  프로젝트를 만듭니다.<br />2.  프로젝트에만 사용 하 여 소스 제어에 추가 \(**파일**, **소스 제어**, **소스 제어에 선택한 프로젝트 추가**합니다.<br />3.  솔루션을 닫습니다.<br />4.  두 개 이상의 프로젝트와 새 솔루션을 만듭니다.<br />5.  소스 제어에 솔루션을 추가 합니다.<br />6.  소스 제어에서 1 단계에서에서 만든 프로젝트를 추가 합니다.<br />7.  메시지가 표시 되는 솔루션의 체크 아웃을 허용 합니다.<br />8.  전체 솔루션에서 확인 합니다.<br />9. 열기는 **소스 제어 변경** 대화 상자입니다.<br />10. \(6 단계\)에서 추가 된 프로젝트를 선택 하 고 클릭 **Unbind**합니다.<br />11. **확인**을 클릭하여 대화 상자를 닫습니다.<br />12. 메시지가 표시 되 면 체크 아웃을 허용 합니다.<br />13. 다시 **소스 제어 변경** 대화 상자입니다.<br />14. \(6 단계\)에서 추가 된 프로젝트를 선택 하 고 클릭 **바인딩할**합니다.<br />15. 원래 위치를 선택 합니다.|솔루션 및 프로젝트 제어 유지 됩니다.|
