# 템플릿을 지 원하는 웹 사이트
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]웹 사이트 프로젝트 템플릿과 항목 템플릿은 새 웹 사이트 프로젝트와 항목을 처음부터 만들 필요가 제거 하 여 개발 프로세스를 가속화 하는 재사용 가능 하 고 사용자 지정 가능한 웹 사이트 프로젝트와 항목 스텁을 제공 합니다.  에 대 한 자세한 내용은 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 서식 파일을 참조 하십시오. [사용자 지정 프로젝트 및 ItemTemplate 만들기](../../ide/creating-project-and-item-templates.md).  
  
## 프로젝트 템플릿 폴더  
 웹 프로젝트 템플릿은 템플릿이 일반적으로 설치 된 *Visual Studio 설치 경로* \\Common7\\IDE\\ProjectTemplates\\Web\\ 각 프로그래밍 언어는 웹 뒤에 이름이 지정 된 하위 폴더에 있습니다.  
  
## 프로젝트 파일  
 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 통합된 개발 환경 \(IDE\) 템플릿 프로젝트 형식에 매핑하는 방법으로 프로젝트 파일 확장명이 있어야 합니다.  웹 프로젝트에 프로젝트 파일이 없으므로이 지원 하기 위해 더미 프로젝트 파일 확장명.webproj가 등록 되어 있습니다.  
  
 언어 기본 설정 웹 프로젝트 시스템을 사용 하는 서식 파일을 필요에 따라 언어 이름 문자열을 추가할 수 있습니다 해당  **새 항목 추가** 대화 상자 항목에 대 한 서식 파일을 기반으로 합니다.  문자열이 파일의 첫 줄이어야 합니다 및 Additemlanguagename에서 Intellisense 엔진 등록에 등록 되는 이름 및 프로젝트 subtype\(vstemplate\)에서 등록 된 이름을 모두 일치 해야 합니다.  자세한 내용은 [웹 사이트의 지원 속성](../../extensibility/internals/web-site-support-attributes.md)를 참조하십시오.  
  
 문자열이 없으면 웹 프로젝트 시스템 프로젝트 서식 파일 서식 파일에는 웹 프로젝트에 추가 하 여 페이지 언어 특성과 파일 확장자에 따라 기본 언어를 확인 하려고 합니다.  
  
## 프로젝트 템플릿  
 웹 사이트 프로젝트 서식 파일에 따라 새 웹 사이트를 구축 하는 데 사용의  **새 웹 사이트** 에 있는  **파일** 메뉴.  세 개의 웹 사이트 프로젝트 형식 현재 지원 됩니다.  
  
-   빈 웹 사이트 프로젝트  
  
-   웹 사이트 프로젝트  
  
-   웹 서비스 프로젝트  
  
### 빈 웹 사이트 프로젝트  
 새 빈 웹 사이트에 따라 이러한 파일을 만들는  **빈 웹 사이트** 가리키는 후 사용할 수 있는 명령을  **새 웹 사이트** 에 있는  **파일** 메뉴:  
  
-   EmptyWeb.vstemplate  
  
     새 빈 웹 사이트를 만드는 과정을 안내 하는 템플릿 파일입니다.  
  
-   EmptyWeb.webproj  
  
     이 파일에서는 프로젝트 템플릿 시스템입니다.  프로젝트 파일 참조에서 EmptyWeb.vstemplate 파일을 충족 합니다.  
  
### 웹 사이트 프로젝트  
 새 웹 사이트에 따라 이러한 파일을 만들는  **ASP.NET 웹 사이트** 을 가리킨 후 사용할 수 있는 명령을  **새 웹 사이트** 에 있는  **파일** 메뉴:  
  
-   Default.aspx  
  
     새 웹 사이트의 기본 홈 페이지입니다.  언어 특성은 코드 숨김 언어를 지정 하 고 CodeFile 특성이이 페이지와 관련 된 코드 숨김 코드를 포함 하는 종속 파일을 지정 합니다.  
  
-   Default.aspx입니다. *확장*  
  
     기본 홈 페이지에 대 한 코드 숨김 코드 포함 된 종속 파일입니다.  코드 숨김 언어 결정의  *확장* 파일.  
  
-   web.config  
  
     루트 web.site 구성 파일입니다.  
  
-   WebApplication.vstemplate  
  
     웹 사이트 솔루션의 내용을 결정 하 고 강제로 App\_Data 폴더 템플릿 파일입니다.  
  
-   WebApplication.webproj  
  
     이 파일에서는 프로젝트 템플릿 시스템입니다.  프로젝트 파일 참조에서 WebApplication.vstemplate 파일을 충족 합니다.  
  
### 웹 서비스 프로젝트  
 새 웹 사이트에 따라 이러한 파일을 만들는  **ASP.Net** 을 가리킨 후 사용할 수 있는 명령을  **새 웹 사이트** 에 있는  **파일** 메뉴:  
  
-   Service.asmx  
  
     새 웹 서비스에 대 한 HTML 페이지입니다.  언어 특성은 코드 숨김 언어를 지정 하 고 코드 숨김 특성은이 서비스와 연결 된 코드 숨김 코드를 포함 하는 종속 파일을 지정 합니다.  
  
-   서비스입니다. *확장*  
  
     서비스 클래스를 구현 하는 종속 파일입니다.  코드 숨김 언어 결정의  *확장* 파일.  
  
-   web.config  
  
-   루트 web.site 구성 파일입니다.  
  
-   WebService.vstemplate  
  
     템플릿 파일 솔루션 웹 사이트의 콘텐츠를 결정 하 고 App\_Code 및 App\_Data 폴더가 생성 됩니다.  서비스입니다. *확장명* 파일은 App\_Code 폴더에 복사 됩니다.  
  
-   WebService.webproj  
  
     이 파일에서는 프로젝트 템플릿 시스템입니다.  프로젝트 파일 참조에서 WebService.vstemplate 파일을 충족 합니다.  
  
## 프로젝트 항목 템플릿 폴더  
 웹 프로젝트 항목 템플릿 서식 파일은 일반적으로 설치 된 *Visual Studio 설치 경로* \\Common7\\IDE\\ItemTemplates\\Web\\ 각 프로그래밍 언어는 웹 뒤에 이름이 지정 된 하위 폴더에 있습니다.  
  
## 프로젝트 항목 템플릿  
 웹 사이트 프로젝트 항목 템플릿을 새 웹 페이지를 웹 사이트에 대 한 응답으로 추가 하는 데 사용의  **기존 항목 추가** 명령입니다.  이러한 종류의 웹 페이지는 현재 지원 됩니다.  
  
-   새 클래스  
  
-   새 HTML 페이지  
  
-   새 Web Form  
  
-   새 마스터 페이지  
  
### 새 클래스  
 이 템플릿은 빈 클래스에에서 정의 된 새 소스 파일을 만듭니다의  **새 클래스 추가** 명령입니다.  
  
-   클래스입니다. *확장*  
  
     빈 클래스를 구현 하는 소스 파일입니다.  코드 숨김 언어 결정의  *확장* 파일.  
  
-   Class.vstemplate  
  
     소스 파일을 만들고 해당 내용을 결정 하는 템플릿 파일입니다.  
  
### 새 HTML 페이지  
 이 서식 파일에 따라 새 웹 페이지를 만듭니다의  **새 HTML 페이지를 추가** 명령입니다.  
  
-   HTMLPage.htm  
  
     시작 웹 페이지의 내용입니다.  이 웹 페이지는 일반적으로 종속 연결 된 코드 숨김 파일이 있습니다.  스마트 페이지를 사용 하는 연결 된 코드 숨김 파일을 만들려면 웹 양식 서식 파일을 대신 사용 합니다.  
  
-   HTMLPage.vstemplate  
  
     웹 페이지를 만들고 그 내용을 확인 하는 템플릿 파일입니다.  
  
### 새 WebForm  
 이 서식 파일에 대 한 응답으로 새 스마트 웹 페이지를 만듭니다의  **새 Web Form을 추가** 명령입니다.  
  
 종속 코드 숨김 소스 파일을 만들려면를 선택  **별도 파일에에서 코드**.  그렇지 않으면 단일 웹 페이지 빈 스크립트 블록 및 없습니다 \< % % \> 만들어집니다. 종속 파일을 연결 하는 지시문입니다.  
  
 선택한 마스터 페이지에 콘텐츠 페이지를 만들려면 선택  **마스터 페이지 선택**.  
  
-   WebForm.aspx  
  
     시작 웹 페이지의 내용입니다.  이 웹 페이지를 종속 연결 된 코드 숨김 파일이 있습니다.  
  
-   WebForm\_cb.aspx  
  
     시작 웹 페이지의 내용입니다.  이 웹 페이지는 연결 된 코드 숨김 종속 파일이 있습니다.  
  
-   코드 숨김 파일입니다. *확장*  
  
     Webform 클래스를 구현 하는 종속 파일입니다.  코드 숨김 언어 결정의  *확장* 파일.  
  
-   ContentPage.aspx  
  
     웹 페이지를 콘텐츠 페이지로 시작 하는 내용입니다.  이 웹 페이지를 종속 연결 된 코드 숨김 파일이 있습니다.  
  
-   ContentPage\_cb.aspx  
  
     웹 페이지를 콘텐츠 페이지로 시작 하는 내용입니다.  이 웹 페이지는 연결 된 코드 숨김 종속 파일이 있습니다.  
  
-   WebForm.vstemplate  
  
     새 웹 페이지 및 해당 종속 파일의 내용이 있는 경우 결정 템플릿 파일입니다.  
  
### 새 마스터 페이지  
 이 서식 파일에에서 새 마스터 페이지를 만듭니다의  **추가 새 마스터 페이지** 명령입니다.  
  
 종속 코드 숨김 소스 파일을 만들려면를 선택  **별도 파일에에서 코드**.  그렇지 않으면 단일 웹 페이지 빈 스크립트 블록 및 없습니다 \< % % \> 만들어집니다. 종속 파일을 연결 하는 지시문입니다.  
  
-   MasterPage.master  
  
     마스터 페이지의 내용을 시작 합니다.  이 마스터 페이지가 종속 연결 된 코드 숨김 파일이 있습니다.  
  
-   MasterPage\_cb.master  
  
     마스터 페이지의 내용을 시작 합니다.  이 마스터 페이지는 연결 된 코드 숨김 종속 파일이 있습니다.  
  
-   코드 숨김 파일입니다. *확장*  
  
     마스터 페이지 클래스를 구현 하는 종속 파일입니다.  코드 숨김 언어 결정의  *확장* 파일.  
  
-   MasterPage.vstemplate  
  
     있는 경우 새 마스터 페이지 및 해당 종속 파일의 내용을 결정 하는 템플릿 파일입니다.
