# <a name="test-area-1-add-toopen-from-source-control"></a>소스 제어에서 열기 /를 추가 하는 테스트 영역 1:
이 소스 제어 플러그 인 테스트 소스 제어에서 프로젝트 또는 솔루션을 배치 하 고 소스 제어에서 검색 영역에 설명 합니다.  
  
## <a name="command-menu-access"></a>명령 메뉴 액세스  
 다음 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 통합된 개발 환경 메뉴 경로 테스트 사례에서 사용 됩니다.  
  
-   에 대 한 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)], 소스 제어에서 열기: **파일**, **열고**, **프로젝트**/**솔루션**; 조회는 에서[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] 위치입니다.  
  
-   다른 소스 제어 플러그 인에 대 한 소스 제어에서 열기: **파일**, **소스 제어**, **소스 제어에서 열기**합니다.  
  
-   소스 제어에 추가: **파일**, **소스 제어**, **소스 제어 파일에 솔루션 추가**, **소스 제어**, **추가 소스 제어에 프로젝트를 선택한**합니다.  
  
-   바로 가기 메뉴 (프로젝트/솔루션) **소스 제어에 솔루션 추가**합니다.  
  
-   소스 제어에서 추가: **파일**, **소스 제어**, **소스 제어에서 프로젝트 추가**합니다.  
  
-   에 대 한 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)], 추가 소스에서 컨트롤은 에서도 사용 가능 **파일**, **추가**, **기존 프로젝트**; 조회에 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] 위치 합니다.  
  
    > [!NOTE]
    >  이 테스트에 로컬 파일이 나 로컬 IIS (웹 서버)의 경로 사용할 수 있습니다.  
  
## <a name="expected-behavior"></a>예상 되는 동작  
  
-   각 지원 되는 프로젝트 형식에 대 한 사용자 "추가" 및 "열 에서" 소스 제어에 있어야 합니다.  
  
-   해당 소스 제어에 프로젝트를 추가 하는 경우 \< *ProjectName*>.vspscc 파일 (프로젝트 힌트 파일)이 생성 됩니다. 제외 파일 목록 및 연결 정보를 포함 합니다. 프로젝트에 대 한 정보가 포함 되어 있으므로이 파일을 삭제 하지 마십시오.  
  
-   해당 소스 제어에 솔루션 추가 될 때 \< *SolutionName*>.vssscc (triple S) 파일이 만들어집니다. 텍스트 파일에는 연결 정보 및 한 프로젝트 힌트 파일과 유사한 제외 파일 목록을 포함합니다. 이 파일은 임시 이므로 소스 제어 데이터베이스에만 존재 합니다.  
  
-   소스 제어에서 솔루션을 열면는 \< *SolutionName*>.vsscc (이중 S) 파일이 소스 제어 데이터베이스에만 있는 임시 파일에 로컬로 만들어집니다. 이 파일은 솔루션 연결 폴더에서 솔루션 파일의 경로 포함합니다. 이 파일은 임시 하 고 "소스 제어에서 열기" 작업이 완료 되 면 로컬 복사본이 삭제 됩니다.  
  
-   프로젝트를 소스 제어에 추가한 후에 (체크 아웃, Get, 및 등)에 모든 소스 제어 작업을 수행할 수 있습니다.  
  
## <a name="test-cases"></a>테스트 사례  
 다음은 추가 대 한 특정 테스트 사례를 / 테스트 영역 소스 제어에서에서 열기.  
  
### <a name="case-1a-add-solution-to-source-control"></a>1a 대/소문자: 소스 제어에 솔루션 추가  
 이 테스트 사례는 소스 제어에 솔루션 추가에 중점을 둡니다.  
  
|작업|테스트 단계|예상된 결과를 확인합니다|  
|------------|----------------|--------------------------------|  
|클라이언트 프로젝트를 소스 제어를 포함 하는 솔루션 추가|1.  클라이언트 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션 추가 (**파일**, **소스 제어**, **소스 제어에 솔루션 추가**).|솔루션/프로젝트 소스 제어에 추가 되었습니다.|  
|파일 시스템 또는 로컬 IIS 웹 프로젝트를 소스 제어를 포함 하는 솔루션 추가|1.  파일 시스템 또는 로컬 IIS 웹 프로젝트를 만듭니다 (찾아보기 단추를 사용 하 여 프로젝트의 위치를 가리키도록 경로 어떤 유형의 웹 프로젝트 작성 결정).<br />2.  소스 제어에 솔루션 추가 (**파일**, **소스 제어**, **소스 제어에 솔루션 추가**).|솔루션/프로젝트 소스 제어에 추가 되었습니다.|  
|원격 사이트 웹 프로젝트를 소스 제어를 포함 하는 솔루션 추가|1.  원격 사이트 웹 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션 추가 (**파일**, **소스 제어**, **소스 제어에 솔루션 추가**).<br />3.  클릭 **확인** FrontPage 액세스 경고 대화 상자에서 합니다.|소스 제어에 솔루션 추가 되었습니다.<br /><br /> 원격 사이트 프로젝트는 소스 제어에 없습니다. (자신의 IIS 서버에서 원격 사이트 프로젝트를 제어할 수 해야 합니다.)|  
|단일 프로젝트 솔루션을 사용 하 여 소스 제어에 추가 **소스 제어에 선택한 프로젝트 추가**합니다.|1.  단일 프로젝트 솔루션을 만듭니다.<br />2.  선택 영역으로 소스 제어에 솔루션만 추가 (**파일**, **소스 제어**, **소스 제어에 선택한 프로젝트 추가**). 이 단계에 성공 하면 다음 단계를 계속 합니다.<br />3.  선택 항목으로 소스 제어에 프로젝트 추가 (**파일**, **소스 제어**, **소스 제어에 선택한 프로젝트 추가**).<br />4.  클릭 **예** 동일한 위치에 프로젝트를 추가 합니다.<br />5.  클릭 **체크 아웃** 에 **편집 하기 위해 체크 아웃** 대화 상자.|`Result from Step 2:`<br /><br /> 프로젝트 및 프로젝트 내에서 모든 파일 체크 아웃 원본 제어 표시기 있고 "에 포함 되지 않은 소스 제어" 도구 설명이 표시 됩니다.<br /><br /> `Result from Step 5:`<br /><br /> 프로젝트 및 솔루션 파일을 소스 제어에에서 동일한 폴더에 있는 합니다.|  
|소스 제어에 솔루션 추가 취소|1.  단일 프로젝트 솔루션을 만듭니다.<br />2.  프로젝트 및 솔루션을 소스 제어에 추가 하려고 시도 합니다. 이 단계에 성공 하면 다음 단계를 계속 합니다.<br />3.  소스 제어 시스템에서 취소 합니다.|`Result from Step 2:`<br /><br /> 프로젝트 위치 소스 설정 컨트롤 대화 상자를 한 번만 나타납니다.<br /><br /> `Result from Step 3:`<br /><br /> 프로젝트 추가 취소 했습니다, 프로젝트/솔루션을 소스 제어 및 모든 여전히 사용할 수 있는 소스 제어 메뉴에 추가 합니다.|  
  
### <a name="case-1b-open-solution-from-source-control"></a>Case 1b 합니다. 소스 제어에서 솔루션 열기

|작업|테스트 단계|예상된 결과를 확인합니다|  
|------------|----------------|--------------------------------|  
|소스 제어에서 클라이언트 프로젝트를 포함 하는 솔루션을 열으십시오|1.  클라이언트 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  솔루션을 닫습니다.<br />4.  새 위치로 소스 제어에서 솔루션을 엽니다.|솔루션/프로젝트를 소스 제어에서 열.|
