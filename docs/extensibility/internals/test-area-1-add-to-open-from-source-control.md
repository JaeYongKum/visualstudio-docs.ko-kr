# 소스 제어에서 열기 /를 추가 하는 테스트 영역 1:
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

이 소스 제어 플러그 인 테스트 솔루션 또는 프로젝트 소스 제어를 배치 하 고 소스 제어에서 검색을 영역에 설명 합니다.  
  
## 명령 메뉴 액세스  
 다음 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 통합된 개발 환경 메뉴 경로 테스트 사례에서 사용 됩니다.  
  
-   에 대 한 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)], 소스 제어에서 열기: **파일**, **열고**, **프로젝트**\/**솔루션**; 조회에 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] 위치 합니다.  
  
-   다른 소스 제어 플러그 인에 대 한 소스 제어에서 열기: **파일**, **소스 제어**, **소스 제어에서 열기**합니다.  
  
-   소스 제어에 추가: **파일**, **소스 제어**, **소스 제어 파일에 솔루션 추가**, **소스 제어**, **소스 제어에 선택한 프로젝트 추가**합니다.  
  
-   바로 가기 메뉴 \(프로젝트\/솔루션\) **소스 제어에 솔루션 추가**합니다.  
  
-   소스 제어에서 추가: **파일**, **소스 제어**, **소스 제어에서 프로젝트 추가**합니다.  
  
-   에 대 한 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)], 추가 소스에서 제어도 있습니다 **파일**, **추가**, **기존 프로젝트**; 조회에 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] 위치 합니다.  
  
    > [!NOTE]
    >  이 테스트를 로컬 파일 또는 로컬 IIS \(웹 서버\)의 경로 사용할 수 있습니다.  
  
## 예상 된 동작  
  
-   각 지원 되는 프로젝트 형식에 대 한 사용자 "추가" 및 "열 에서" 소스 제어 수 있어야 합니다.  
  
-   해당 소스 제어에 프로젝트를 추가 하는 경우 \<*ProjectName*\>.vspscc 파일 \(프로젝트 힌트 파일\)이 생성 됩니다. 제외 파일 목록 및 연결 정보를 포함 합니다. 프로젝트에 대 한 정보가 포함 되어 있으므로이 파일을 삭제 하지 마십시오.  
  
-   소스 제어에 해당 하는 솔루션은 추가 하는 경우 \<*SolutionName*\>.vssscc \(triple S\) 파일이 만들어집니다. 텍스트 파일에는 연결 정보 및 한 프로젝트 힌트 파일과 유사한 제외 파일 목록을 포함합니다. 이 파일은 일시적 이며 소스 제어 데이터베이스에만 존재 합니다.  
  
-   소스 제어에서 솔루션을 열면는 \<*SolutionName*\> 소스 제어 데이터베이스에만 존재 하는.vsscc \(이중 S\) 파일은 임시 파일에 로컬로 만들어집니다. 이 파일에서 솔루션 파일을 솔루션 연결 폴더에서 경로 포함 합니다. 이 파일에는 일시적 이며 "소스 제어에서 열기" 작업이 완료 되 면 로컬 복사본 삭제 됩니다.  
  
-   프로젝트를 소스 제어에 추가한 후 \(체크 아웃, Get, 및 등\)에 모든 소스 제어 작업을 수행할 수 있습니다.  
  
## 테스트 사례  
 다음은 추가 대 한 특정 테스트 사례 \/ 테스트 영역 소스 제어에서에서 열기.  
  
### 1a case: 소스 제어에 솔루션 추가  
 이 테스트 사례는 소스 제어에 솔루션 추가에 중점을 둡니다.  
  
|작업|테스트 단계|예상된 결과 확인 하려면|  
|--------|------------|-------------------|  
|소스 제어에 클라이언트 프로젝트를 포함 하는 솔루션을 추가 합니다.|1.  클라이언트 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션 추가 \(**파일**, **소스 제어**, **소스 제어에 솔루션 추가**\).|솔루션\/프로젝트 소스 제어에 추가 되었습니다.|  
|파일 시스템 또는 로컬 IIS 웹 프로젝트 소스 제어에 포함 된 솔루션을 추가 합니다.|1.  파일 시스템 또는 로컬 IIS 웹 프로젝트를 만듭니다 \(찾아보기 단추를 사용 하 여 프로젝트의 위치를 가리키도록 경로 어떤 유형의 웹 프로젝트를 만들지를 결정\).<br />2.  소스 제어에 솔루션 추가 \(**파일**, **소스 제어**, **소스 제어에 솔루션 추가**\).|솔루션\/프로젝트 소스 제어에 추가 되었습니다.|  
|소스 제어에 원격 사이트 웹 프로젝트를 포함 하는 솔루션을 추가 합니다.|1.  원격 사이트 웹 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션 추가 \(**파일**, **소스 제어**, **소스 제어에 솔루션 추가**\).<br />3.  클릭 **확인** FrontPage 액세스 경고 대화 상자에서입니다.|소스 제어에 솔루션 추가 되었습니다.<br /><br /> 원격 사이트 프로젝트는 소스 제어에 없습니다. \(자신의 IIS 서버에서 원격 사이트 프로젝트를 제어할 수 해야 합니다.\)|  
|단일 프로젝트 솔루션을 사용 하 여 소스 제어에 추가 **소스 제어에 선택한 프로젝트 추가**합니다.|1.  단일 프로젝트 솔루션을 만듭니다.<br />2.  선택 영역으로 소스 제어에 솔루션만 추가 \(**파일**, **소스 제어**, **소스 제어에 선택한 프로젝트 추가**\). 이 단계에 성공 하면 다음 단계를 계속 합니다.<br />3.  선택 항목으로 소스 제어에 프로젝트 추가 \(**파일**, **소스 제어**, **소스 제어에 선택한 프로젝트 추가**\).<br />4.  클릭 **예** 같은 위치에 프로젝트를 추가 합니다.<br />5.  클릭 **체크 아웃** 에서 **편집 하기 위해 체크 아웃** 대화 상자입니다.|`Result from Step 2:`<br /><br /> 프로젝트 및 프로젝트 내에서 모든 파일 체크 아웃 소스 제어 표시기 있고 "하지 소스 제어" 도구 설명이 표시 됩니다.<br /><br /> `Result from Step 5:`<br /><br /> 프로젝트 및 솔루션 파일은 소스 제어에 동일한 폴더에 있습니다.|  
|소스 제어에 솔루션 추가 취소 합니다.|1.  단일 프로젝트 솔루션을 만듭니다.<br />2.  소스 제어에 프로젝트 및 솔루션을 추가 하려고 했습니다. 이 단계에 성공 하면 다음 단계를 계속 합니다.<br />3.  소스 제어 시스템에서 취소 합니다.|`Result from Step 2:`<br /><br /> 프로젝트 위치 소스 설정 제어 대화 상자는 한 번만 나타납니다.<br /><br /> `Result from Step 3:`<br /><br /> 프로젝트 추가 취소 했습니다, 프로젝트\/솔루션을 소스 제어 및 모든 여전히 사용할 수 있는 소스 제어 메뉴에 추가 합니다.|  
  
### Case 1b 합니다. 소스 제어에서 솔루션 열기  
 이 테스트 사례는 소스 제어에서 솔루션 열기에 중점을 둡니다.  
  
|작업|테스트 단계|예상된 결과 확인 하려면|  
|--------|------------|-------------------|  
|소스 제어에서 클라이언트 프로젝트를 포함 하는 솔루션을 엽니다|1.  클라이언트 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  솔루션을 닫습니다.<br />4.  새 위치에 소스 제어에서 솔루션을 엽니다.|솔루션\/프로젝트 소스 제어에서 열립니다.|  
|로컬 또는 소스 제어에서 IIS 웹 프로젝트를 포함 하는 솔루션을 엽니다|1.  로컬 또는 IIS 웹 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  솔루션을 닫습니다.<br />4.  새 위치에 소스 제어에서 솔루션을 엽니다.|솔루션\/프로젝트 소스 제어에서 열립니다.|  
|소스 제어에서 원격 사이트 웹 프로젝트를 포함 하는 솔루션을 엽니다|1.  원격 사이트 웹 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다. 이 단계에 성공 하면 다음 단계를 계속 합니다.<br />3.  솔루션을 닫습니다.<br />4.  새 위치에 소스 제어에서 솔루션을 엽니다.|`Result from Step 2:`<br /><br /> 원격 사이트 웹 소스 제어에 있지 않습니다.<br /><br /> `Result from Step 4:`<br /><br /> 소스 제어에서 솔루션 열기<br /><br /> 원격 사이트 프로젝트가 로드 되 면 소스 제어에서 것만 합니다.|  
  
### 사례 1 c: 소스 제어에서 솔루션을 추가 합니다.  
 이 테스트 사례는 소스 제어에서 솔루션 추가에 중점을 둡니다.  
  
|작업|테스트 단계|예상된 결과 확인 하려면|  
|--------|------------|-------------------|  
|빈 솔루션에 추가\-단일 프로젝트 솔루션|1.  단일 프로젝트 솔루션을 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  솔루션을 닫습니다.<br />4.  두 번째 빈 솔루션을 만듭니다.<br />5.  소스 제어에서 이전에 제어 솔루션 추가 \(**파일**, **소스 제어**, **소스 제어에서 프로젝트 추가**\).|추가 된 프로젝트에 표시 **솔루션 탐색기** 및 체크 인 합니다.|  
|단일 프로젝트와 솔루션에 추가\-단일 프로젝트|1.  단일 프로젝트와 솔루션을 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  솔루션을 닫습니다.<br />4.  두 번째 빈 솔루션을 만듭니다.<br />5.  소스 제어에서 이전에 제어 솔루션 추가 \(**파일**, **소스 제어**, **소스 제어에서 프로젝트 추가**\).|추가 된 프로젝트에 표시 **솔루션 탐색기** 및 체크 인 합니다.|  
|솔루션에 추가\-솔루션을 선택 하 여 소스 제어에 추가 합니다.|1.  프로젝트와 솔루션을 만듭니다.<br />2.  선택 항목으로 소스 제어에 솔루션만을 추가 합니다. 이 단계에 성공 하면 다음 단계를 계속 합니다.<br />3.  솔루션을 닫습니다.<br />4.  새 솔루션을 만듭니다.<br />5.  소스 제어에서 이전에 제어 솔루션 추가 \(**파일**, **소스 제어**, **소스 제어에서 프로젝트 추가**\).|`Result from Step 2:`<br /><br /> 프로젝트는 소스 제어에 없습니다.<br /><br /> `Result from Step 5:`<br /><br /> 솔루션 항목을 첫 번째 해결 하는 경우 추가할 수는 없습니다 소스 제어에서 나타나지 않습니다.<br /><br /> 첫 번째 솔루션에서 프로젝트를 사용할 수 없는 상태로 나타납니다.|
