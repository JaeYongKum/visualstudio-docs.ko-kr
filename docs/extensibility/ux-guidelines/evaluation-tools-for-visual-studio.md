# Visual Studio 용 평가 도구
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

## Visual Studio 용 장인 검사 목록  
 이 검사 목록을 사용 하 여 visual 및 상호 작용 세부 정보에 대 한 사용자 환경을 품질을 평가 합니다.  
  
### 개요  
  
-   모든 명령을 결과로 피드백에 알려주는 사용자는 자신의 명령을 단계가 수행 되었음을 확인 합니다.  
  
-   모든 UI 요소 및 컨트롤 모든 테마에 고대비 모드에 표시 되는지 확인 합니다.  
  
-   비활성을 확인 하 고 표준 및 고대비 모드 둘 다에서 현재 선택 영역을 항상 구분 합니다.  
  
-   포커스가 표시 되 고 명백한 항상 인지 확인 합니다.  
  
### 성능  
  
-   명령에서는 1 초 이상 완료 하는 경우 몇 가지 종류의 "사용 중" 표시기가 표시 되는지 확인 합니다.  
  
-   명령을 완료 하는 데는 명시적 진행률 표시줄 하거나 10 초 이상 걸리는 경우 확인 비활성화 상태 \(기본 설정\) 인지 또는 비활성화, 표시 됩니다.  
  
### UI 텍스트  
  
-   모든 레이블이 문장 또는 대문자로 하 고 텍스트가 없는 완전히 소문자 인지를 확인 합니다.  
  
    ||수정|잘못 된|  
    |-|--------|----------|  
    |**명령 텍스트 \(모두\)**|문장 경우:<br /><br /> **디렉터리 이름:**|디렉터리 이름:|  
    |**단추 텍스트 \(클라이언트\)**|제목 경우:<br /><br /> **\[기본 설정\]**|SET AS DEFAULT로|  
    |**단추 텍스트 \(온라인\)**|문장 경우:<br /><br /> **\[기본값으로 설정\]**||  
  
-   그룹 머리글 및 단추를 제외한 모든 레이블에 %를 쌍으로 있는 컨트롤 앞에 확인 합니다.  
  
-   단추, 명령 및 사용자 입력을 캡처하는 UI를 실행 하는 명령 링크는 줄임표 \(...\)에 종료 확인 **\[...\]**합니다.  
  
     예를 들면 다음과 같습니다.  
  
    -   **\[고급...\]** 대화 상자에서 단추입니다.  
  
    -   도구 메뉴에서 명령 옵션 \(**도구 \> 옵션**\) 명령의 의도 자체 대화 상자를 시작 하기 때문에 줄임표 \(...\)를 가져올 해야 합니다.  
  
-   업계 표준 용어를 제외 하 고 없는 약어를 UI에 포함 되어 있는지 확인 합니다. 예를 들어, HTML 아니고 TCP\/IP 해야 명시 되어야 OOM \(메모리 부족\)와 PII \(개인 식별이 가능한 정보\) 해야 하는 경우.  
  
### 키보드 액세스  
  
-   키보드와 각 작업을 수행 하는 방법을 확인 합니다. 각 컨트롤에 대 한 키보드 액세스를 통해이 작업을 수행 하는 일반적으로 이지만 일부 시각적 영역에 대 한 코드 뷰로 이동 등 문제를 해결 하려면 허용 합니다.  
  
-   논리적 순서 \(왼쪽에서 오른쪽 및 위쪽에서 아래쪽\)에서 컨트롤을 통해 Tab 수 있는지 확인 합니다. 이 대부분의 컨트롤에 대 한 모범 사례, 모든 컨트롤에는이 방법은 필요 합니다. 예를 들어 컨트롤은 단일 탭 정지를 갖는 그룹에 해당 라디오 단추를 확인 합니다.  
  
-   모든 컨트롤에 레이블을 반드시 각 레이블에 니모닉에 있는지 확인 \(예외 레이블이 지정 된 컨트롤의 탭에 나오는 일부 비 레이블이 지정 된 컨트롤을 포함 하는 데 사용\).  
  
-   니모닉 충돌이 없는지 확인 합니다.  
  
### 글꼴  
  
-   모든 글꼴 \(글꼴, 크기, 색\) 일관 되 게 사용 되는 계층 구조를 유지 하는지를 확인 합니다.  
  
-   모든 UI 요소 환경 글꼴 서비스를 사용 하는지 확인 합니다. \(참조 [글꼴 및 Visual Studio에 대 한 서식 지정](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md)\)  
  
     서비스가 사용 되 고 있는지를 확인 하려면로 이동 **도구 \> 옵션 \> 글꼴 및 색**합니다. 설정 드롭다운 목록에서 환경 글꼴을 선택 하 고 \(예: Harrington 또는 만화 San\) stylistically 다른 것에 글꼴을 변경 12pt로 크기를 설정 합니다. 확인을 클릭 합니다. IDE를 다시 시작 해야 하지만 대부분 UI 즉시 변경 됩니다. 영역을 다시 시작에도 글꼴 변경 내용이 반영 하지는 환경 글꼴을 사용 하지 않습니다.  
  
-   글꼴 \(예: 굵게 또는 확장 된 텍스트\) 서비스의 파생 클래스에는 크기 및 환경 글꼴 크기가 변경 되 면 "일반" 텍스트와 관련 하 여 서식 유지를 확인 합니다.  
  
-   글꼴 확대 인해 클리핑 버그가 없는 한지 확인 합니다. 클리핑됩니다 글꼴 가능성이 고정된 높이 컨트롤 또는 고정된 높이 컨테이너의 결과로 생성 됩니다.  
  
### 대화 상자  
  
-   확인 대화 상자 제목 시작 명령과 동일 합니다.  
  
-   모든 표준 컨트롤이 운영 체제와 일치 되는지 확인: 배경색은 표준 및 컨트롤이 표준 컨트롤에서 다르게 나타날 수 있는 특별 한 re 템플릿 스타일 있어야 합니다.  
  
-   폼 내의 여백 12 픽셀 이어야를 균일 하 고 일관적인 나타나야 확인 합니다.  
  
-   대화 상자에 생성 되는 창 또는 IDE 셸 내에서 가운데 맞춤 나타나는지 확인 합니다.  
  
-   유용 하 게 하는 경우에 대화 상자 크기를 조정할 수 있어야 합니다. 크기를 조정할 수 있는 대화 상자는 크기 조정 시 적절 한 컨트롤 크기를 조정 해야 대화의 다른 부분 일정 하 게 유지 하는 동안 확인 합니다.  
  
-   크기 조정 가능한 대화 상자 \(크기, 위치, 대화 상자 컨트롤 및 등의 확장\) 모든 사용자 조정 크기를 유지 하도록 확인 합니다.  
  
-   제목 표시줄에 아이콘이 있는지 확인 하십시오.  
  
-   제목 표시줄에 최소화, 최대화 단추가 없습니다 한지 확인 합니다.  
  
#### 대화 상자 작업 단추 \(VS 클라이언트에만 해당\)  
  
-   작업 단추이 순서에 있는지 확인: **확인**, **취소**, **적용**합니다.  
  
-   확인 **확인** 및 **취소** 단추는 표준 크기: 75 x 23 \(픽셀\)입니다.  
  
-   확인 **확인** 및 **취소** 문자열 길이 관계 없이 동일한 크기의 단추는 합니다.  
  
-   작업 단추 레이블 표준 보다 넓은 되도록 단추를 필요로 하는 경우를 확인 하는 해당 **취소** 동일한 크기의 단추는 합니다.  
  
-   단추와 연결 된 컨트롤 사이의 6 픽셀 여백 인지 확인 합니다.  
  
-   확인는 **확인** 및 **취소** 단추 니모닉 \(선택 키 밑줄이 그어진된 문자에 정의 된\)를 갖지 않습니다.  
  
-   확인 단추를 하나 \(일반적으로 **확인**\)는 기본적으로 포커스를 가집니다.  
  
-   확인 **Esc** 대화 상자를 취소 합니다.  
  
-   확인 **Enter** Enter를 처리 하는 컨트롤에 포커스가 없으면 기본 단추를 실행 합니다.  
  
-   있는지는 **확인** 및 **취소** 단추 대화 상자의 오른쪽 아래에 배치 됩니다. 거의 예외 오른쪽 위에 있는 세로로 쌓을 수 하는 것이 좋습니다.  
  
-   대화 상자 내에서 가로 맞춤에 다른 단추가 있는 경우에 세로 구성이 사용 되는지 확인 합니다.  
  
### 컨트롤 표준  
  
#### 일반  
  
-   가능한 경우 좋은 기본값은 속도 사용자 상호 작용 및 결과 안전 또는 일반적인 방향으로 직접 사용자를 확인 합니다.  
  
-   작동 하 여 표준 컨트롤 같은 방식으로 사용자가 이전 경험을 기반된 어떻게 알 수 있도록 확인 합니다.  
  
#### 레이블 컨트롤  
  
-   각 컨트롤에는 레이블 및 각 레이블에 시각적으로 쌍을 이루는 \(일반적으로 4\-6 픽셀 범위\) 내에서 해당 컨트롤 및 해당 하는 컨트롤 보다 다른 컨트롤에 더 가까운를 확인 합니다.  
  
-   레이블이 플러시 배치 됩니다 확인 컨트롤과 왼쪽 위에 배치 하는 경우 왼쪽 가장자리와 왼쪽에 배치 하는 경우 레이블의 초기 입력된 텍스트의 기준으로 정렬 되도록 수평적으로 가운데 맞춤 합니다.  
  
-   여러 누적된 레이블 및 입력된 컨트롤에는 컨트롤의 왼쪽에 배치 되어, 경우 레이블을 왼쪽 맞춤은 그리고 오른쪽 및 컨트롤에서 같은 간격에 하지 대화 상자의 가장자리에서 균등 한 거리는 플러시 확인 하십시오. 추가 간격 그룹화를 나타내기 위해 필요한 경우가 아니면 쌍 균등 하 게 배포 되어야 합니다.  
  
#### 입력된 컨트롤 \(텍스트 상자 및 콤보 상자\)  
  
-   기본 환경 글꼴을 사용 하는 경우 텍스트 상자, 콤보 상자 및 단추에 대 한 디스플레이 높이 23 하는 모든 픽셀을 확인 합니다.  
  
-   힌트 텍스트를 사용 하는 경우 색으로 설정 되어 있는지 확인 하십시오 `Environment.ControlEditHintText` 색 서비스를 사용 합니다.  
  
-   필드가 이와 같이 식별 해야 하는 필수 필드가 경우 확인 합니다.  
  
    -   배경으로 설정 된 `Environment.ControlEditRequiredBackground` 전경으로 설정 됩니다 `Environment.ControlEditRequiredHintText`  
  
    -   로 표시 되는 컨트롤 내에서 힌트 텍스트 임을 **"\< Required \>"**  
  
#### 단추 컨트롤  
  
-   긴 텍스트를 수용 하지 않는 한 단추는 75 x 23 픽셀의 최소 크기를 확인 합니다.  
  
-   단추를 떠난를 오른쪽 3\-5 픽셀으로 콘텐츠에 대 한 안쪽 여백을 확인 합니다.  
  
-   줄임표만 작은 사각형 단추를 사용 하는 것이 좋습니다 **\[...\]** 대신에 **\[찾아보기...\]** 단추 \(또는 이와 유사한 기능\). 사용 하는 경우 단추 크기가 23 x 23 인지 확인 합니다.  
  
-   여러 개 있으면 **\[찾아보기...\]** 되어 있는지 확인 한 다음 대화 상자에서 단추 단축된 버전 \(줄임표 전용 **\[...\]**\) 모두에 사용 됩니다.  
  
-   해당 줄임표 확인 **\[...\]** 단추 니모닉 필요는 없습니다. 옆에 있는 입력된 컨트롤에 포커스가 있을 때 탭 줄임표 \(...\) 단추에 포커스를 이동 해야 합니다.  
  
-   확인 단추, 명령 및 명령 링크 추가 사용자 입력을 캡처하는 보조 UI를 시작 하는 줄임표 \(...\)에서 끝나야 **\[...\]**합니다.  
  
#### 하이퍼링크  
  
-   Hyperlink 컨트롤에 되지 활성 상태인 경우에 빨간색 깜박이 확인 합니다. 이 컬러 서비스 사용 되 고 있지 않습니다.  
  
-   사용 되는 VS 색 인지 확인 합니다.  
  
    -   `Environment.ControlLinkText`  
  
    -   `Environment.ControlLinkTextHover`  
  
    -   `Environment.ControlLinkTextPressed`  
  
-   하이퍼링크는 단락에 포함 하지 않는 한 밑줄 없이 파란색 나타나는지 확인 합니다.  
  
#### 확인란  
  
-   여러 줄 텍스트를 포함 하는 확인란, 모든 줄에 걸쳐 세로로 가운데 텍스트의 첫 번째 줄 맞춰 상자를 확인 합니다.  
  
-   있는지 확인란 항상 이진 항목을 선택 하 고 사용자를 이동 하거나 열지 않습니다 새 창 또는 페이지를 확인 합니다.  
  
-   확인란 입력된 컨트롤에 관련 된 옵션을 표시, 배치 되었음을 플러시 왼쪽과 밀접 해당 컨트롤의 관계를 나타내기 위해 아래를 확인 합니다.  
  
-   확인 확인란 **하지** 대화 상자 또는 페이지의 전체 내용을 사용 하는 수단으로 사용 합니다.  
  
#### 그룹 상자  
  
-   확인 대화 상자는 대화의 전체 콘텐츠를 포함 하는 단일 그룹 상자 내에 포함 되지 않습니다.  
  
-   각 그룹 상자 내에서 두 개 이상의 컨트롤 한지 확인 합니다.  
  
-   거의 대화 상자에서 두 개 이상의 그룹 상자 있을 해야 합니다.  
  
-   중첩된 그룹 상자 없이 한지 확인 합니다.  
  
### 아이콘  
  
-   어두운 테마에서 아이콘 반전 올바르게 나타나는지 확인 합니다.  
  
-   모든 아이콘은 핵심 개념을 기반으로 한다는 것을 확인 합니다.  
  
-   각 아이콘은 distinct, 쉽게 인식할 수를 두 개 이상의 개념 \(없이 언어\/상태 한정자\)는 확인 합니다.  
  
-   기본 아이콘 공간 내 가운데 나타나는지 확인 합니다.  
  
-   모든 아이콘 고대비 모드에서 읽을 수 나타나는지 확인 합니다.  
  
-   사용 되는 모든 색 색 사용 표준 부합을 확인 합니다.  
  
-   아이콘 주위 없는 후광 \(테두리\) 한지를 확인 합니다. \(있는 경우의 무리가 일치 해야 인접 한 UI의 배경색\) 합니다.  
  
### UI 터치 사용
