# Visual Studio에 대 한 애니메이션
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

## 애니메이션 기본 사항  
  
### Visual Studio에서 애니메이션 모범 사례  
 Visual Studio의 IDE 전체에서 일관 되 고 사용자에 게 친숙 하는 애니메이션 스타일을 유지 하기 위해 이러한 규칙을 따릅니다.  
  
-   **주의 기울여야 합니다.** 애니메이션 특정 용도로 사용 되는 것을 제한 합니다.  
  
-   **타이밍 및 속도 중요 한** 전환 신속 하 고 자연 스러운 느낌을 확인 하려면:  
  
    -   1\/2 초 \(500 밀리초\) 내에서 애니메이션된 전환을 완료 합니다.  
  
    -   애니메이션 자주 발생 하는 사용자의 워크플로 중단 하지 시킬 만큼 빠른 해야 합니다.  
  
    -   따라서 빠르거나를 이해 하기 어려운 아니라 하지 속도가 느린 수행 하는 한 전환을 완료 하기 위해 잠시 기다리십시오 임을 jarring 애니메이션 되지 않아야 합니다.  
  
    -   변수 타이밍 중요성을 강조 하기 위해 사용 합니다. 예를 들어 클래스 다이어그램에 있는 항목의 시퀀스를 탐색할 때 항목 간의 전환을 통해 속도 다음 중요 한 항목에 초점을 느리게 합니다.  
  
-   **점진적 비선형 감속\/가속을 사용 하 여** 다른 한 상태에서 진정 하 고 자연 스러운 이동의 의미를 제공 합니다.  
  
-   가능 하면 **가리키기 미묘한 애니메이션을 사용 하 여** 마우스에서 대화형 요소를 나타냅니다.  
  
-   의존 하는 경우 과도 하 게 애니메이션에 해당 기능에서 다음 **설정을 해제 하는 수단을 제공** 로컬로 \(기능에 대 한 모든 사용자\)에 옵션으로는 **도구 \> 옵션** 대화 합니다.  
  
-   **한 번에 하나의 애니메이션 이루어져야** 하나만 정보의 알아볼 수 있습니다.  
  
-   **주의 해야 하는 것이 중요 합니다.** 대부분의 경우에서 애니메이션의 용도 제공 하 요청 시 사용자의 주의 필요가 없습니다. 타이밍, 순서 지정 기능 및 동작에 미묘한 변화 인식에 크게 영향을 줄 수 및 효과적이 고 비효율적인 애니메이션으로 만들 수 있습니다.  
  
-   무언가에 관심을 모으기 위해 애니메이션을 사용 하는 경우 **것이 좋다는 사용자를 중단 해야**의 필자의 생각 합니다.  
  
-   **진행률 또는 상태를 표시 하는 경우** 애니메이션을 통해:  
  
    -   기본 프로세스는 진행 되지 않으면 때 진행률 이동을 표시를 중지 합니다.  
  
    -   비활성화 프로세스에서 결정 되지 않은 프로세스를 구분 합니다.  
  
    -   애니메이션 식별이 가능한 완료 및 오류 상태에 있는지 확인 합니다.  
  
    -   상태를 표시 하 고 실제 사용의 추가 정보를 제공 하 여 실제 값을 갖는지 확인 하는 효과 애니메이션의 사용을 최소화 합니다. 예로 일시적인 상태가 변경 되 고 긴급 한 문제  
  
#### 하지 않습니다.  
  
-   작은 움직임 \(작은 공간에 이동\)를 사용 하 여, 페이드 및 개체를 이동 하는 동안 변경 우선적으로 사용 합니다.  
  
-   화면 부동산의 큰 영역을 통해 수행 하는 애니메이션을 사용 합니다. 크기에 관계 없이 애니메이션의이 스타일은 사용자에 게 불필요 합니다.  
  
-   현재 포커스가 사용자 개체 또는 상호 작용 하는 애니메이션을 사용 합니다.  
  
-   사용자가 깜박임 중지 있도록 깜박이 알림에 응답 하는 등 상태를 다시 설정 하려면 사용자가 개입 해야 하는 애니메이션을 사용 합니다. 어떤 방식에서으로 상호 작용 하 고 해제 충분 해야 합니다.  
  
 이러한 모범 사례에 대 한 응용 프로그램에 대 한 자세한 내용은 참조 하십시오. [애니메이션 패턴](../../extensibility/ux-guidelines/animations-for-visual-studio.md#BKMK_AnimationPatterns)합니다.  
  
### 애니메이션 메트릭  
  
-   시스템 보다 작은 10 밀리초에서 사용자 제스처를 시각적으로 응답 해야 합니다.  
  
-   애니메이션된 전환을 완료 하려면 500 밀리초 보다 오래 걸리지 않습니다.  
  
-   긴 시간을 필요로 하는 전환에 대 한 보정을 위해 한 가지 방법은; 두 부분으로 구분 하는 예를 들어 애니메이션의 첫 번째 부분에 다음 콘텐츠 서서히 사라지는 \(최대 500 밀리초\) 컨테이너를 빈 콘텐츠 컨테이너 \(최대 500 밀리초\) 수 있습니다.  
  
-   계산할 수 있는 부하 시간을 결정 하는 진행률 표시기 \(%\-완료 진행률 표시기\) 선호 됩니다.  
  
-   로드 시간을 계산할 수 없는, 커서 또는 \(로드 또는 작업 표시기\) 포함 된 회전 애니메이션 등 사용 중인 표시기는 적합 합니다.  
  
### Communicator 애니메이션  
 Visual Studio UI에서 애니메이션 통신 도구로 작동합니다.  다양 한 UI;의 구조 변경 등의 정보를 통신 하는 데 사용 됩니다. 예를 들어 때 메뉴가 열립니다 하거나 닫습니다. 애니메이션 설치 진행률 시각화 같은 복잡 한 시스템의 시간에 따라 변하는 작동 상태를 가시화 하는 데 도움이 또는 경고 및 알림 들의 관심을 갖게 하기 위한 사용할 수 있습니다.  
  
 UI 애니메이션은 일반적으로 네 가지 방법으로 작동 합니다: 시각화, 관심을 갖게, 시뮬레이션 및 응답을 표시 시간\/진행 상황입니다.  
  
#### 시각화  
 애니메이션 개체의 3 차원 특성을 강조 하 고 해당 공간 구조를 시각화 하는 사용자에 대 한 보다 쉽게 수 있습니다. 이를 위해 애니메이션 수 있습니다 필요 전체 원에 있는 개체를 회전 느린 것를 주고받는 또는 가까이 개체 가져오기 켜고 약간 롤오버 또는 포커스를 강조 하는 크기를 늘릴 합니다.  
  
 3d 개체 \(프로그래밍 방식으로 또는 수동으로\) 디자이너 미리 결정 해야 하는 사용자 컨트롤로 이동할 수 수 있지만 어떻게 최적의 애니메이션 개체의 최적의 이해를 제공 하는 이동 됩니다. 이 애니메이션 수 프로그래밍 다음 사용자 제어 이동을에서는 개체를 조작 하는 방법을 이해 하도록 설정 해야 하는 반면, 개체 위에 커서를 배치 하 여 사용자가 활성화할 수 있습니다. 단일 축 또는; 한 번에 한 방향에 대 한 이동을 제한합니다 중 하나 크기를 조정, 회전 또는 변환, 있지만 동시에 둘 이상의 수행 하지 마십시오.  
  
 시각화 범주에는 데이터, 관계, 상태의 측면에는 구조, 시퀀스 및 시간입니다.  
  
##### 데이터  
 복잡 하 고 변수 정보를 보여 줍니다.  
  
-   차트 및 그래프와 같은 정보 시각화를 통해 이동  
  
-   순서를 통해 단계별 실행, 둘러보기 및 페이징  
  
-   세부 정보를 파악해을 특정 정보를 강조 표시  
  
-   세부 정보 및 추가 정보는 포커스가 있는 요소 위에 오버레이  
  
-   다른 한 구조 또는 조직 표현에서 모핑  
  
-   시간 슬라이더, 작업 우주선 바퀴 및 전송 컨트롤 \(재생, 중지 및 일시 중지\)을 사용 하 여 시간이 지남에 따라 변경에 따라 합니다.  
  
##### Relationships  
  
-   지정 된 항목과 관련 된 항목 또는 항목 간의 관계를 보여 줍니다.  
  
-   계층 및 부모\-자식 또는 형제 관계 표시  
  
-   다른 요소 생성  
  
-   하나의 요소가 다른 요소에 최소화  
  
-   다른 테더 링 된 하나의 요소  
  
##### 상태  
  
-   콘텐츠 업데이트 합니다.  
  
-   사용자 포커스 및 선택  
  
-   진행률  
  
-   오류  
  
##### 구조  
  
-   한 노드에서 구조 피벗  
  
-   방향을 새로 설정  
  
-   최소화 하 고, 최대화 또는 확장 및 축소  
  
##### Sequence  
  
-   슬라이드 쇼 시퀀스  
  
-   그림을 통해 대칭 이동  
  
##### 시간  
  
-   시간 경과 시간과 스크린 캐스트에 대 한 표시 변경  
  
-   삭제, 이동 실행 취소, 다시 실행  
  
-   기록 상태를 복원 합니다.  
  
#### 관심을 갖게  
 목표가 여러에서 단일 요소에 대 한 사용자의 관심을 모으기 위해 또는 업데이트 된 정보를 사용자에 게 경고할 경우 애니메이션 적합할 수 있습니다. 예를 들어, 응용 프로그램 시작 페이지 페이지가 로드 된 후에 슬라이드 시작 단추를 사용할 수 있습니다.  
  
 일반적으로 화면에서 이동 마지막 요소의 동안 사용자의 주의 유도 합니다.  일련의 애니메이션된 요소에서 사용자의 주의 마지막 개체가 이동 될 예정입니다.  
  
##### 경으십시오  
  
-   사용자 경고, 주의 확보 하 고, 진행률 표시  
  
-   작업 중인 했는지 올바르게 또는 잘못 표시 또는 진행 중이거나 진행 중인 변경  
  
-   온라인으로 자세한 정보를 찾기 또는 현재 작업에 대 한 학습 하는 등의 작업 중 사용자에 게 확인  
  
##### 알림  
  
-   오류가 발생 하는 방법에 대 한 사용자 경으십시오  
  
-   사용자에 게 표시 하는 경우 하 고 싶은 다른 일에 참석 인터럽트  
  
-   조심 스럽게 프로세스가 완료 되거나 다운로드 완료 되 면 같은 변경 하는 사용자를 게 알립니다.  
  
#### 시뮬레이션  
 이 범주 여실히 차원에 적용 됩니다.  
  
-   개체에서 발생 한 위치 또는 어디를 보여 줍니다.  
  
-   확장 및 축소 또는 열기 및 닫기  
  
-   팬, 스크롤 및 페이지 설정  
  
-   스택 및 z 순서 지정  
  
-   회전식 및 accordion  
  
-   대칭 이동 및 회전 UI  
  
#### 응답 및 진행률 표시기  
 진행률 표시기의 큰 장점은 몇을 가지 있습니다.  
  
-   모두 비활성화 상태 및 부정형 진행률 표시기에는 시스템에서 충돌이 발생 했습니다 하 고 문제에 대해 작업 중인 사용자에 원할 것입니다.  
  
-   비활성화 상태 표시기 작업까지 진행 되었는지를 파악할 진행 되 고, 완료에 더 가깝게 얻는 과정을 거칠 뿐만 아니라 사용자에 게 제공 합니다.  
  
##  <a name="BKMK_AnimationPatterns"></a> 애니메이션 패턴  
  
### 개요  
 Visual Studio에서 애니메이션 특정 기능을 제공 하 고 사용자의 생산성을 방해 하지 되어야 합니다. 포함을 적용 하기 위해 일반 애니메이션 특성:  
  
-   한 눈에 띄지 않는  
  
-   자연스럽 고 현실적인  
  
-   정교 하 고 게 당한 후  
  
-   빠르고 효율적으로  
  
-   완화 되지 hurried  
  
 다음 그림에는 Visual Studio에서 사용 하도록 권장 되는 애니메이션 스타일 보여 줍니다. 정교한 애니메이션 없고 애니메이션 같은 페이드 인\/페이드 아웃 가장 자주 사용 됩니다. 와 같은 확장 및 축소 애니메이션 이동의 제한 된 응용 프로그램이, X 및 Y 위치가 변경 및 회전 합니다.  
  
 ![Recommended animation styles for Visual Studio](../../extensibility/ux-guidelines/media/1202-a_vsanimstyles.png "1202\-a\_VSAnimStyles")  
  
 **Visual Studio에 권장되는 애니메이션 스타일**  
  
#### 표시 하거나 숨길  
 이 패턴을 사용 하면 요소 뷰 아웃 하 고 전환 애니메이션 없이 다시 표시에서 전환합니다.  
  
 ![Appear&#47;Disappear animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-b_appearanddisappear.png "1202\-b\_AppearAndDisappear")  
  
##### 올바른 사용  
 즉시 나타나거나 사라질 사용자는 무시 되거나 방해 필요로 하는 새로운 UI 요소입니다. 또한 느린 이동 애니메이션 및 사라집니다 표시 스타일으로도 발생 하지 않습니다 성능 끌어 옵니다도 인식 될 수 있습니다.  
  
##### 잘못 된 사용  
 UI가 변경 된 내용, 갑자기 사용자에 게 알지 나타납니다 하 고 상황에 맞는 이해 도움이 되는 애니메이션 추가 없는 경우.  
  
##### 애니메이션 속성  
 시간 지연을 일반적으로 0 초입니다.  
  
##### 예제  
  
-   도구 창 자동 숨기기  
  
-   매개 변수 도움말 및 IntelliSense와 같은 UI 편집기에 키보드 활성화  
  
-   확장\-축소 코드 영역  
  
#### 전파가  
 이 패턴을 사용 하면 UI 요소는 표시 되지 않습니다 \(불투명도 0%\)에서 전환 표시 \(100% 불투명\) 또는 그 반대의 경우:  
  
 ![Fade&#45;in&#47;Fade&#45;out animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-c_fadeinfadeout.png "1202\-c\_FadeInFadeOut")  
  
##### 올바른 사용  
 이 UI 애니메이션에서 가장 일반적으로 좋습니다. 흐름을 중단 하지 않고 관심을 추가 하는 미묘한 효과 합니다. 일부 경우에 사용자 조차 모를 수 있다고이 애니메이션 및 단순히 한 부드러운, 이동을 UI 시스템을 인식 합니다.  
  
##### 애니메이션 속성  
  
-   불투명도 시작: 페이드 인에 대 한 0%, 100% 페이드 아웃  
  
-   끝 불투명도: 페이드 인 100%, 페이드 아웃에 대 한 0%  
  
-   기간: 200 밀리초 독립 실행형으로 100 밀리초 조합 애니메이션 시퀀스의 일부로 사용 될 때  
  
-   스타일 감속\/가속: 사인 inout을 각각  
  
##### 예제  
  
-   도구 창 자동 숨기기  
  
-   메뉴 열기 및 닫기  
  
-   배경 및 전경 탭 전환

 이 패턴을 사용 하면 UI 요소 색상 A에서에서 색으로 변경 b:

##### 올바른 사용  
 으로 UI 요소를 하나의 컨텍스트 또는 상태에서 다른 색이 변경 될 때 애니메이션된 전환 됩니다.  
  
##### 애니메이션 속성  
  
-   시작 색: 특정 UI  
  
-   끝 색: 특정 UI  
  
-   기간: 200 밀리초 독립 실행형으로 100 밀리초 조합 애니메이션 시퀀스의 일부로 사용 될 때  
  
-   스타일 감속\/가속: 사인 inout을 각각  
  
##### 예제  
  
-   문서 창 상태 전환 \(활성, 활성 및 비활성 마지막\)  
  
-   도구 창 상태 전환 \(초점과 포커스가 없는\)  
  
#### 확장 및 축소  
 이 패턴을 사용 하면 UI 요소는 X, Y 또는 두 방향에서 확장 됩니다.  
  
 ![Expand&#47;Contract animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-e_expandcontract.png "1202\-e\_ExpandContract")  
  
##### 올바른 사용  
 으로 UI 요소에서 다른 특정 컨텍스트에서 크기를 변경 하면 애니메이션된 전환 됩니다.  
  
##### 애니메이션 속성  
  
-   X 배율: % 또는 특정 차원 \(픽셀 단위\)  
  
-   Y 비율: % 또는 특정 차원 \(픽셀 단위\)  
  
-   앵커 위치: \(왼쪽에서 오른쪽 언어\)에 대 한 왼쪽 위 또는 \(오른쪽에서 왼쪽 언어\)에 대 한 오른쪽 위 일반적으로  
  
-   기간: 200 밀리초 독립 실행형으로 100 밀리초 조합 애니메이션 시퀀스의 일부로 사용 될 때  
  
##### 예제  
  
-   아키텍처 탐색기 패널 확장 및 축소  
  
-   시작 페이지 항목 확장 및 축소
