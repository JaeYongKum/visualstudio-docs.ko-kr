# <a name="visual-studio-team-services-vsts-benefit-in-visual-studio-subscriptions"></a>Visual Studio 구독의 VSTS(Visual Studio Team Services) 혜택

## <a name="overview"></a>개요 
모든 언어에 대한 무료 Git 리포지토리, 민첩한 계획 도구 및 호스트된 빌드 등으로 IDE를 완벽하게 보완합니다. 
- 팀을 위한 완벽한 개발 환경을 만듭니다. 
- 아이디어부터 릴리스까지 지속적으로 관리합니다. 
- 코드 품질을 향상하고 조기에 문제를 파악합니다.
- Azure 배포를 자동화 및 간소화합니다.
- 강력한 기능으로 생산성을 개선합니다.

어떤 구독을 사용하고 있는지 확실하지 않나요?  전자 메일 주소에 할당된 모든 구독을 보려면 [https://my.visualstudio.com/subscriptions](https://my.visualstudio.com/subscriptions?wt.mc_id=o~msft~docs)에 연결하세요. 일부 구독이 표시되지 않으면 하나 이상이 다른 전자 메일 주소에 할당되어 있을 수 있습니다.  해당 구독을 보려면 해당 전자 메일 주소로 로그인해야 합니다. 

Visual Studio 구독을 활성화하는 데 사용한 것과 동일한 ID를 사용하여 VSTS에 로그인하면 자동으로 인식됩니다. 여기에는 구독자 포털에 로그인할 때 사용하는 기본 ID 및 사용자가 Visual Studio 구독에 구성한 대체 ID가 모두 해당됩니다. VSTS는 Microsoft 계정(예: @outlook.com) 및 회사 또는 학교 계정(귀사에서 관리하는 Azure Active Directory 사용)을 모두 지원합니다. VSTS에서 기본 ID와 대체 ID를 모두 사용하여 원하는 만큼의 VSTS 계정을 멤버로 조인할 수 있습니다.

VSTS를 사용하려면 계정이 필요합니다. 기존 계정으로 로그인하거나 새 계정을 만들 수 있습니다.  새 계정을 만들려면 다음을 수행합니다.
1.  [https://my.visualstudio.com/benefits](https://my.visualstudio.com/benefits?wt.mc_id=o~msft~docs)에 로그인합니다.
2.  도구 섹션에서 Visual Studio Team Services 타일을 찾고 혜택 타일 아래쪽에서 “시작” 링크를 클릭합니다.   

이러한 VSTS 기능은 다음 구독에 포함됩니다. 
- Visual Studio Enterprise: [기본](https://www.visualstudio.com/team-services/compare-features/), [테스트 관리자](https://marketplace.visualstudio.com/items?itemName=ms.vss-testmanager-web), [패키지 관리](https://marketplace.visualstudio.com/items?itemName=ms.feed)
- Visual Studio Professional: [기본](https://www.visualstudio.com/team-services/compare-features/)
- MSDN 플랫폼: [기본](https://www.visualstudio.com/team-services/compare-features/), [테스트 관리자](https://marketplace.visualstudio.com/items?itemName=ms.vss-testmanager-web)
- Visual Studio Test Professional: [기본](https://www.visualstudio.com/team-services/compare-features/), [테스트 관리자](https://marketplace.visualstudio.com/items?itemName=ms.vss-testmanager-web)

3.  VSTS 프로젝트 사이트의 이름을 입력합니다.  

4.  **Git** 또는 **TFVC(Team Foundation 버전 제어)**를 사용하여 프로젝트를 관리할지 여부를 선택합니다.  선택한 버전 제어 시스템은 생성된 각 팀 프로젝트에 영구적으로 적용되지만, 동일한 팀 프로젝트 컬렉션에서 TFVC 및 Git 팀 프로젝트를 둘 다 사용할 수도 있습니다.  어느 것을 사용할지 확실하지 않나요? 
    - Git: Git는 분산 버전 제어 시스템입니다. 각 개발자마다 자신의 개발 컴퓨터에 전체 원본 리포지토리의 복사본이 있습니다. 개발자는 자신의 개발 컴퓨터에서 각 변경 내용 집합을 커밋하고 네트워크 연결 없이 기록 및 비교와 같은 버전 제어 작업을 수행할 수 있습니다.  Git에 대해 [자세히 알아보세요](https://www.visualstudio.com/docs/git/gitquickstart).
    - TFVC: TFVC(Team Foundation 버전 제어)는 중앙 집중식 버전 제어 시스템입니다. 일반적으로 팀 멤버는 자신의 고유 개발 컴퓨터에 각 파일 버전 하나만 보유합니다. 기록 데이터는 서버에만 보관됩니다. 분기는 경로에 기반을 두며 서버에서 만들어집니다. Team Foundation 버전 제어에 대해 [자세히 알아보세요](https://www.visualstudio.com/docs/tfvc/overview).

 
5.  **변경 세부 정보**를 클릭하여 프로젝트 이름, 작업을 구성하는 방법(Agile, Scrum, CMMI), 프로젝트를 호스팅할 수 있는 위치 및 다른 사람들과 작업을 공유하는 방법에 대한 옵션을 사용자 지정합니다.  "계속"을 클릭합니다.

# <a name="create-your-vsts-account"></a>VSTS 계정 만들기

6.  계정을 만드는 데 잠시 시간이 걸리며, 지정한 이름을 사용하여 첫 번째 프로젝트에 대한 VSTS 페이지가 표시됩니다.  이제 Visual Studio Team Services를 사용할 준비가 되었습니다!

또한 계정이 성공적으로 만들어졌는지 확인하는 전자 메일을 받게 됩니다.  계정 URL, 로그인 및 기본 설정된 이메일 주소도 나열됩니다.

## <a name="faq"></a>FAQ
### <a name="q--what-is-the-difference-between-free-basic-access-and-advanced-access"></a>Q: 무료, 기본 액세스 및 고급 액세스의 차이점은 무엇인가요?
A: 사용 가능한 기능은 액세스 수준에 따라 다릅니다.  [액세스 수준](/vsts/security/access-levels)에 대한 자세한 정보를 찾으세요. 

### <a name="q--how-do-i-add-users-to-my-vsts-account-or-team-project"></a>Q: 사용자를 VSTS 계정 또는 팀 프로젝트에 추가하려면 어떻게 할까요?
A: VSTS에서 [사용자 및 액세스를 관리](/vsts/accounts/add-account-users-from-user-hub)하는 방법을 알아보세요.
